## 1. 배경

ADR-008 시크릿 보안 위반(case-sensitive guard bypass), WS 가짜 데이터 fabrication,
OrderBookProvider 미구현 등 보안·데이터 무결성 결함 발견. 두 개의 fix 브랜치 생성·커밋됨.

## 2. Fix 브랜치 현황

### `fix/bithumb-ws-guards`
| 항목 | 내용 |
|---|---|
| `ws_secret_guard.py` | header/payload key 비교를 `.lower()` 정규화 → case-insensitive guard |
| `ws_mapping.py:107` | ticker 필수 price/volume 필드 누락 → `Decimal("0")` fabrication 제거, `SchemaMismatchError` raise |
| `ws_mapping.py:129` | orderbook delta unknown orderType → ask silently 처리 제거, `SchemaMismatchError` raise |
| `ws_mapping.py` | non-dict orderbook entry silently drop → `SchemaMismatchError` raise |
| `rest_throttle.py:53` | `in_flight_count()` 에서 lock 없이 deque evict 제거 → read-only count |
| `exceptions.py` | `PublicOnlyViolationError(BithumbApiError)` 계층 수정 |

### `fix/bithumb-client-adapter`
| 항목 | 내용 |
|---|---|
| `client.py` | REST forbidden header 비교 lowercase frozenset → case-insensitive |
| `client.py` | retry: `TimeoutException+ConnectError` → `httpx.TransportError` 상위 클래스 |
| `adapter.py` | 비0000 Bithumb envelope → `SchemaMismatchError` 제거, `BithumbApiError` raise |
| `adapter.py` | `BithumbOrderBookProvider` 신규 구현 (REST orderbook endpoint → `OrderBookModel`) |
| `ws_client.py` | `_last_ts: dict[str, datetime]` 추가 → timestamp 기반 dedup, 재연결 시 초기화 |
