## §7 설계 서사

**관찰 1 — Daemon 부재**: `mctrader-market-bithumb` 의 candle 관련 code 는 `BithumbCandleProvider`
(adapter.py) 의 thin REST wrapper 만 — backtest / backfill eager fetch 용도. **candle polling
collector daemon 은 본 repository 에 부재**. ADR-026 §D6 의 "daemon 중지" 절차는 N/A —
이미 retire 된 상태로 간주. Provider 자체는 ADR-026 §D1 (legacy immutable SSOT) 따라 영구 보존.

**관찰 2 — cutoff guard 위치**: legacy candle write 의 단일 entry point = `mctrader-data` CLI
`backfill` 명령 (`get_candle_provider → write_candles`). 본 명령에 cutoff guard 를 추가하면
post-cutoff candle Parquet 침투를 단일 지점에서 차단 — Bithumb WS transaction subscriber
(Story-4 MCT-138) 와 충돌 없음. `--allow-post-cutoff` escape hatch 로 DR 시나리오 보존.

**관찰 3 — Provenance helper-only**: ADR-009 §D16 provenance column 은 ohlcv schema 본체에
추가 의무이나, 본 Story 는 helper API (`assign_provenance`) + cutoff config 만 박제. Schema-
level write 박제는 별도 Story (Story-3 / Story-5 implementation seal) 의 영역으로 분리.

**관찰 4 — Cross-repo 의존성 부재**: data PR + market-bithumb PR 이 동일 module 을 공유하지
않으므로 sequential merge 의무 없음. Story 지시 따라 data → market-bithumb 순으로 진행
했으나 병렬 merge 도 가능.
