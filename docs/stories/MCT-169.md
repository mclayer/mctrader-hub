# MCT-169 — L1 NAS verify + immediate local delete + tier promotion path 재구성 (EPIC-tier-promotion-single-source Story-3)

## §1 동기

**EPIC-tier-promotion-single-source Story-3** — ADR-029 D3=C + D10=A owner Story. L1 NAS HEAD verify 후 grace 0 즉시 local delete + ambiguity invariant violation enforcement (D10=A). L2/L3 source read = NAS GET stream (INV-3, D3=C 정합).

### 1.1 직접 동기

MCT-168 (L1 NAS DualWriter wiring, D1+D2) LAND 후 단계. NAS = single source of truth 확립의 핵심 단계:
1. **D3=C (NAS HEAD verify + grace 0)**: L1 NAS HEAD verify (ETag + VersionId 또는 byte-match) 통과 즉시 local L1 delete. 24h grace 폐기. ambiguity 즉시 차단.
2. **D10=A (ambiguity invariant)**: NAS+local 동시 존재 = violation. `verify_no_ambiguity(segment_id)` 검증 함수 + pytest fixture 의무.
3. **L2/L3 source = NAS GET**: L2Compactor.compact_hour() + L3Compactor.compact_day() 의 소스 read = local Path → NAS boto3 Range GET stream. INV-3 확립.

### 1.2 D3+D10 결정 (ADR-029 박제분)

- **D3=C**: Local delete = NAS HEAD verify + grace 0 (immediate after verify). Researcher↔PMO 충돌 해소: version/etag 검증으로 24h grace 대체, ambiguity 즉시 차단.
- **D10=A**: ambiguity 차단 = invariant violation enforcement. NAS+local 동시 존재 = violation. codeforge-plugin#620 Fix-1 의 production evidence gate 정합.

## §2 Phase 0 Context (brainstorm 답습)

PMOAgent 2nd pass 박제분 (별 세션, 2026-05-14).

### 2.1 AC (8)

- **AC-1 promotion.py NAS HEAD verify**: `s3.head_object` → ETag + VersionId 또는 Content-Length byte-match 시에만 promotion proceed
- **AC-2 immediate local delete (grace 0)**: HEAD verify pass 즉시 `Path.unlink(missing_ok=False)`, time.sleep 0
- **AC-3 ambiguity invariant**: `verify_no_ambiguity(segment_id)` — NAS HEAD success ∧ local exists = violation
- **AC-4 L2/L3 source = NAS**: `l2.py.compact_l1_to_l2` + `l3.py.compact_l2_to_l3` source = boto3 get_object Range ranged GET, local Path open 0
- **AC-5 get_streaming inverse**: `nas_storage/get_streaming.py` 또는 `nas_uploader.py` extend — Range header ranged GET
- **AC-6 D7 reader cache placeholder**: `compactor/reader_cache.py` 인터페이스 stub (MCT-170 구현)
- **AC-7 HEAD mismatch**: ETag mismatch ∨ 404 = `PromotionVerifyError` raise, local delete 미실행
- **AC-8 invariant test pytest fail**: `tests/integration/compactor/test_ambiguity_invariant.py` — NAS+local 동시 존재 fixture 시 fail

### 2.2 INV (6)

- **INV-1 SoT exclusivity**: ∀ segment, `nas_exists ⊕ local_exists` = true (XOR, D10=A)
- **INV-2 grace = 0**: HEAD verify pass → local unlink 사이 wall-clock < 100ms
- **INV-3 NAS = SoT for L2/L3**: L2/L3 source byte stream = NAS GET, local fallback 0 (D3=C)
- **INV-4 verify-before-delete atomicity**: HEAD verify fail = local 유지 (partial state 차단)
- **INV-5 version match**: version-enabled bucket VersionId 일치 강제 (available 시)
- **INV-6 promotion idempotency**: local 부재 + NAS 존재 segment 재호출 = no-op

### 2.3 Risk (3)

- **R-1 HIGH**: L2/L3 read latency 증가 (local 50us → NAS LAN 2-5ms). 완화: MCT-170 reader cache (D7) prerequisite, AC-6 placeholder 선행
- **R-2 MED**: HEAD verify race (S3 strong consistency 2020 이후 보장, MinIO 동일). 완화: AC-7 retry 1회 (50ms backoff) 후 fail
- **R-3 MED**: invariant test flakiness. 완화: post-promotion 정합 검증으로 한정

### §5.1 mctrader-data: promotion.py (신규)

**파일: src/mctrader_data/compactor/promotion.py**

```
class PromotionVerifyError(Exception):
    """HEAD verify fail — local delete 미실행 (INV-4)"""

class PromotionResult:
    status: Literal["promoted", "already_promoted", "verify_failed"]
    segment_id: str
    nas_key: str
    local_path: Path | None

def promote_l1(
    *,
    local_path: Path,
    nas_uploader: NASUploader,
    segment_id: str,
) -> PromotionResult:
    """L1 NAS HEAD verify → immediate local delete (D3=C, grace 0).

    Steps:
    1. NAS HEAD verify (AC-1):
       - head_object(Key=nas_key) → ETag + VersionId match (INV-5)
       - retry 1회 (50ms backoff, R-2 mitigation, AC-7)
    2. verify_no_ambiguity(segment_id): NAS exists ∧ local exists → raise (AC-3, D10=A)
    3. HEAD verify PASS → Path.unlink(missing_ok=False) (AC-2, INV-2 wall-clock < 100ms)
    4. local 부재 + NAS 존재 → PromotionResult(status="already_promoted") (INV-6)

    Raises:
        PromotionVerifyError: HEAD 404 or ETag/VersionId mismatch (AC-7, INV-4)
    """

def verify_no_ambiguity(
    *,
    segment_id: str,
    nas_uploader: NASUploader,
    local_path: Path,
) -> None:
    """AC-3 / D10=A: NAS+local 동시 존재 = violation.

    NAS HEAD success ∧ local_path.exists() → raise AmbiguityViolation
    (invariant 위반 — MCT-169 INV-1 SoT exclusivity)
    """
```

```python
"""reader_cache.py — D7 reader cache placeholder (MCT-169 AC-6).

from __future__ import annotations
from pathlib import Path
from typing import IO, Protocol

class ReaderCache(Protocol):
    """D7 reader cache interface (MCT-170 구현)."""
    def get(self, nas_key: str) -> IO[bytes] | None: ...
    def put(self, nas_key: str, data: bytes) -> None: ...
    def invalidate(self, nas_key: str) -> None: ...

class NullReaderCache:
    """No-op cache — MCT-169 placeholder. MCT-170 이 실 LRU cache 로 대체."""
    def get(self, nas_key: str) -> IO[bytes] | None:
        return None
    def put(self, nas_key: str, data: bytes) -> None:
        pass
    def invalidate(self, nas_key: str) -> None:
        pass
```
