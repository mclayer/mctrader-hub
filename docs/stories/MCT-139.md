## §7 설계 서사

### 구현 결정 (DeveloperPL dispatch 결과)

- **Storage layout 정합**: Story가 명시한 `market/ticks/schema_version=tick.v1.1/`는 logical schema. 실제 L1 compactor 가 박제하는 Hive layout은 `market/transaction/schema_version=tick.v1/tier=L1/exchange=.../symbol=.../date=.../node=.../part-<run_id>.parquet`. Cold resampler는 후자 사용.
- **tick.v1 vs tick.v1.1 schema gap**: storage가 v1.0 (8 cols)을 persist, aggregation core 는 v1.1 (11 cols)을 요구. Cold adapter가 deterministic `trade_id` 합성 `{exchange}:{symbol}:{ts_iso}:{seq}` + default `is_taker=True` / `validation_status=OK` 로 boundary 가교. **Open question forwarded to ArchitectPL** (§12 참조).
- **Date partition 미사용**: WAL ingester가 `time.time()` wall-clock 으로 partition `date=` 결정 → 데이터의 `ts_utc.date()` 와 분리. 정확성 위해 `date` 술어 사용하지 않음 — `ts_utc` row-level filter + Parquet footer statistics 가 file-level skipping 담당.
- **임의 timeframe 처리**: closed Timeframe enum (1m/5m/15m/1h/4h/1d) 밖의 seconds(13/47/300/...)는 `InformationBarModel(bar_label="time_<seconds>")` 로 노출. CandleModel은 closed enum만 지원.
- **DuckDB TimeZone pinning**: `SET TimeZone='UTC'` 명시 — KST host에서 `TIMESTAMP WITH TIME ZONE` 컬럼이 `Asia/Seoul` tzinfo로 반환되어 `UTCDateTime` validator 거부 발생. session-level UTC pinning + Python-side `astimezone(utc)` defence-in-depth.
- **Hot/Cold SSOT**: DuckDB와 Polars 양 엔진이 Story-3 aggregators 를 직접 import — DuckDB UDF 우회. 엔진은 partition pruning + sort + iter; algorithm은 Story-3 가 SSOT. `test_polars_matches_duckdb_*` 가 byte-identical output 확인.

### 아키텍처 다이어그램

```
Cold path query (DevPL implementation)
  caller (engine backtest / web Streamlit / reconciliation)
     │
     ▼
  DuckDBResampler.resample_time / resample_information_bar
     │
     ├── DuckDB (SET TimeZone='UTC'; read_parquet hive_partitioning=true)
     │     symbol=? + exchange=?(opt) + ts_utc range filter, ORDER BY ts_utc
     │
     ▼
  Iterator[TickRowV1_1]  (v1.0 → v1.1 boundary adapter: synth trade_id)
     │
     ▼
  mctrader_data.aggregation.core (Story-3 SSOT)
     ├── TimeBarAggregator
     ├── VolumeBarAggregator
     ├── TickBarAggregator
     └── DollarBarAggregator
     │
     ▼
  Iterator[InformationBarModel | CandleModel(source="transaction_derived")]
```

`PolarsResampler` 는 identical pipeline — DuckDB 자리를 `polars.scan_parquet(hive_partitioning=True).filter().sort()` 로 대체.

## §12 Open Questions (forwarded to ArchitectPL)

1. **tick.v1.1 storage upgrade**: 현재 L1 compactor는 tick.v1 (8 cols)만 persist. v1.1 (11 cols, `trade_id` / `is_taker` / `ingest_seq` / `payload_hash` / `validation_status`)은 logical schema. Cold adapter가 synthesise하는 `trade_id` 는 deterministic이지만 source 그대로 보존 의무가 있다면 storage upgrade 필요.
2. **`date` Hive partition 의미 재정의**: 현재 WAL ingest wall-clock 기반 → reconcile 시 데이터 `ts_utc.date()` 가 더 자연스러움. Compactor 가 lineage-time vs event-time partition 둘 다 박제할지 ADR 결정 필요.
3. **DuckDB SQL `time_bucket()` 직접 사용**: 현재는 Hot/Cold SSOT 우선 → Story-3 core post-fetch routing. 향후 50 sym × N year scan 시 throughput 위해 SQL-side time-bucket 으로 path bifurcate 옵션 (with consistency 검증 harness)이 필요할 수 있음.
