    """

        self,
        endpoint: str,                   # NAS_MINIO_ENDPOINT (e.g., http://nas.local:9000)
        access_key: str,
        secret_key: str,
        bucket: str = "mctrader-market", # ADR-027 D1 정합
        retry_queue: RetryQueue | None = None,
        metrics: PrometheusExporter | None = None,
    ) -> None:
        # _client_lock = threading.Lock() — _get_client() lazy init thread-safety (P1-3 박제, FIX#2)
        ...

    def _get_client(self) -> Any:
        """Lazy boto3 client init under threading.Lock (P1-3 박제, FIX#2).

        MCT-152 dual-write window 진입 시 multi-worker (compactor + dual_write_window_runner)
        가 동시 첫 호출 시 race → 두 client 동시 생성 + 한쪽 GC 누락 risk → Lock 으로 차단.
        Single-worker 환경에서는 lock contention 0 (lazy init 1회만).
        """

        self,
        key: str,
        data: bytes | Path,
        *,

        suppress_enqueue: bool = False,  # FIX#2 P0-2 박제 — drain() 호출 시 True

            "uploaded",                # 신규 PUT 성공 (object 영속화 완료)

#### §6.2.2 `RetryQueue` (nas_storage/retry_queue.py) — FIX#2 P0-1 Option A 재정의 (quarantine = retry-with-backoff)

**FIX#2 P0-1 boundary 재정의 (Sonnet decider Option A 채택, 2026-05-13)**:

기존 wording "row count > 10000 시 oldest segment quarantine + alert" 가 quarantine semantics 미정의 →
양쪽 reviewer (Codex + Claude) 가 동일 P0 해석 = "drop semantics" → **S8 RPO=0 위반 + NFR-3 손실 0 위반 + §11.5 byte-level identity 위반**.

**새 정의 (Option A — quarantine = retry-with-backoff, NEVER drop)**:

- **quarantine = drop 아님**. SQL `state` enum `["pending", "in_flight", "quarantined", "succeeded"]` 4-state.
  `quarantined` = 정상 drain cycle 에서 일시 격리된 상태 — **별 quarantine drain cycle (longer backoff schedule)** 로 별도 retry.
- **enqueue() invariant 박제**: enqueue 호출은 **절대 segment drop 0**. RPO=0 (S8 user_confirmed) 정합.

**FIX#4 P1-NEW-2 박제 — enqueue() 진입 흐름의 직교 2-stage guard (Option A 사용자 confirmed 2026-05-13)**:

`enqueue()` 진입 시 **2-stage check 가 직교적으로 분리** — 상호 의존 0, 순서 고정:

- **Stage 1 (unconditional hard_floor guard, FIX#4 박제)**:
  enqueue() 진입 직후 **첫 단계** = `pending + quarantined >= hard_floor_segments` 체크.
  - **threshold_breached (max_segments) 와 무관**한 **unconditional guard** — `if threshold_breached:` block **외부** 에 위치 의무.
  - hard floor 도달 시 → **segment 영속화 0** + `EnqueueResult.status="hard_floor_blocked"` 즉시 반환 + SOP MANUAL_GATE signal emit.
  - 본 가드 conditional 내부에 위치 시 (FIX#3 P1-NEW-2 회귀, FIX#3 impl `if total_active >= self.hard_floor:` 가 `if threshold_breached:` block **내부** 위치): threshold_breached 산정 logic 에 따라 quarantined seg heavy case 에서 hard floor skip 가능. 예: pending=5000 + quarantined=10000, max_segments=10000 (threshold_breached 가 specific 산정 path 의존 — pending-only count 또는 다른 sub-state filter 시 False) → hard floor check skip → total=15001 도달 가능. 일반화: hard_floor 와 threshold_breached 산정 source 가 다르거나 threshold_breached 의 inner state 가 hard_floor 와 직교한 어떤 case 든 → bounded backlog NFR 위반 + AC-4 MANUAL_GATE escalation false negative. **불변 해결책 = guard 의 conditional dependency 0 (Stage 1 위치 강제)**.
  - **불변식**: pending + quarantined ≤ hard_floor_segments (합계 hard upper bound). 본 invariant 는 enqueue() 진입 흐름 내 모든 분기에서 보존.
- **Stage 2 (conditional threshold demotion, P0-1 박제 보존)**:
  Stage 1 통과 후 → `pending + quarantined >= max_segments OR bytes_used >= max_bytes` (threshold breach) 체크.

  - threshold_breached 미발동 시 → 신규 segment 정상 pending enqueue.
  - 본 stage 는 **quarantine 강등 trigger only** — hard floor enforcement 와 직교.

- **bounded floor (진짜 가득 차는 경우)**: Stage 1 의 unconditional guard 가 enforce — pending + quarantined > 10000 도달 차단 (도달 직전 enqueue 부터 hard_floor_blocked return).
  ADR-027 D5 hot path 무영향 invariant 보존 — collector WAL/L1 침범 0, compactor only pause.
- **quarantine drain schedule**: exponential backoff `[1m, 5m, 30m, 2h]`. 매 attempt last_attempt_ts column 갱신.

        **2-stage 직교 guard 의무 (FIX#4 P1-NEW-2 박제, Option A 사용자 confirmed 2026-05-13)**:

        Stage 1 (unconditional hard_floor guard) — enqueue() 진입 직후 첫 단계, threshold 와 무관:

                emit_sop_signal("hard_floor_blocked")

            threshold_breached = (pending+quarantined >= max_segments) or (bytes_used >= max_bytes)

                emit_alert(...)
                status = "enqueued_with_quarantine_demotion"
            else:
                status = "enqueued"
            persist_segment(...)  # 모든 정상 enqueue 는 disk write
            return EnqueueResult(status=status, ...)

        **EnqueueResult.status enum**:
        - "enqueued": 정상 pending 등록 (threshold 미발동).
        - "enqueued_with_quarantine_demotion": threshold 발동, oldest pending 강등 + 신규 정상 pending 등록.

        - 본 invariant 는 §11.5 byte-level identity + S8 RPO=0 + ADR-027 D5 hot path 무영향 의 1차 ledger guarantee.

    def drain(self, uploader: NASUploader) -> DrainStats:
        """2-cycle FIFO drain (FIX#2 P0-1 + P0-2 박제).

        Cycle 1 — pending drain (정상 backoff):
            SELECT WHERE state='pending' ORDER BY enqueue_ts LIMIT batch_size
            → state='in_flight' UPDATE → uploader.put(..., suppress_enqueue=True) 호출 (P0-2 박제).
            결과 분기:
              - PutResult.status="uploaded" or "skipped_idempotent" → state='succeeded' UPDATE → row 삭제 (또는 batch GC).

            SELECT WHERE state='quarantined' AND (NOW - last_attempt_ts) >= backoff_schedule[attempt_count]

              - PutResult.status="uploaded" or "skipped_idempotent" → state='succeeded' (quarantine 해제).

        DrainStats = {pending_drained, quarantined_drained, pending_failed, quarantined_failed,
                       hard_floor_blocked, sop_signals}. 모든 실패 case 의 segment 는 disk retain (drop 0).

        AC-2 mapping: persistence verify."""
