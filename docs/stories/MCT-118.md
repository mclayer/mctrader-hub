```powershell
# ADR-019 D2/D3/D5: agent 환경 부트스트랩 표준
# 사용법: . scripts/agent-preflight.ps1 -RepoPath <path> -Branch <branch>

param(
    [string]$RepoPath = ".",
    [string]$Branch = "",
    [switch]$UseWorktree
)

# D2: Python 버전 검증
$pyVersion = py -3.12 --version 2>&1
if ($LASTEXITCODE -ne 0) { throw "py -3.12 not found" }

# D3: Editable install
py -3.12 -m pip install -e $RepoPath --quiet

# D4: Branch 검증
if ($Branch) {
    $current = git -C $RepoPath branch --show-current
    if ($current -ne $Branch) { throw "Branch mismatch: expected $Branch, got $current" }
}
```

### 11.2 기술 노트 (PMO 감사 관점)

**T1. D3 PowerShell 문자열 보간 버그 — 실행 중 발견·수정**
초안 스크립트의 `"$resolvedPath[dev]"` 패턴이 PowerShell 문자열 보간 규칙상
`$resolvedPath` 변수의 `[dev]` 인덱서로 해석되어 `null`을 반환. 정정 패턴
`$resolvedPath + "[dev]"` (또는 `"$($resolvedPath)[dev]"`) 적용. 향후 ps1
스크립트 작성 시 동일 함정 주의 — pytest fixture 가 PowerShell 토큰
인터폴레이션 규칙을 자동 검증하지 않으므로 lint 보강 후보.

**T2. ADR이 해결하려는 문제가 스토리 실행 중 재현됨 (메타 사건)**
본 스토리 작업 중 다른 세션이 동시에 main에서 작업 → branch switch race
발생, `git checkout feat/mct-118-preflight-scripts` 재수행 필요. ADR-019 D1
worktree 의무가 구현 완료 후 즉시 본 스토리 자체에 적용되었어야 함. 의도치
않은 dogfooding 사례 — 차기 스토리부터 D1 worktree 적용 의무 발효.
