## §1 사용자 요구사항 (verbatim)

Epic MCT-112 Story-11 — Dual-write reconciliation harness (2-4주 검증 + Hot/Cold consistency).

**범위**: cross-repo (`mctrader-data` + `mctrader-engine`). Phase 5 (순차, Story-7/8/9 완료 후 진입).

### 변경 내용

**Dual-write Harness** (`mctrader_data/reconciliation/dual_write_harness.py`):
- 2-4주 dual-write 기간 legacy candle vs transaction-derived bar 비교
- daily random sample 1-5% 또는 100 symbol-contract window/day
- edge-case fixtures: threshold 정확 boundary, time bar inclusion edge, KRW rounding edge

**Hot/Cold Consistency Harness** (`mctrader_data/reconciliation/hot_cold_consistency.py`):
- Story-3 Aggregation Core Lib 의 same algorithm, different driver 검증
- drift % 측정 — SLO < 0.01% bar count mismatch (ADR-025 §D5, Risk 2 mitigation)
- 위반 시 strategy 입력 Cold fallback trigger

**Strategy reproducibility report** (engine 측):
- 동일 strategy + 동일 기간 backtest 를 legacy vs derived 양쪽 실행 → PnL diff report
- Risk 3 mitigation (dual-write 기간 strategy 재현성)
- Story-9 의 `SignalProvenanceLog` JSONL cross-reference

### §8 Test Contract (drift SLO green gate)

- drift % < 0.01% 조건 미달 시 fail-closed
- SLO 박제 ADR-025 (Aggregation Core Lib Contract) reference

## §7 설계 서사

DualWriteHarness + HotColdConsistencyHarness + edge_case_fixtures (data) + StrategyReproducibilityReporter (engine) 의 4 컴포넌트가 Story-12 retirement gate 의 evidence pack 을 구성한다. 모든 모듈은 Story-3 Aggregation Core Lib (ADR-025 §determinism) 의 same-algorithm-different-driver 패턴을 그대로 답습 — Hot path (Story-8 StreamingAggregator) 와 Cold path (Story-5 DuckDBResampler) 모두 동일한 aggregator instance 를 driver 로 호출하므로 byte-identical 출력이 contract 보장. 본 Story 의 verifier 는 이 invariant 가 production 환경에서도 유지되는지 daily sample + drift SLO < 0.01% 로 측정.

핵심 설계 결정:
- **drift SLO gate 는 fail-closed** — `ConsistencyDriftError` 가 throw 되면 retirement 진행 불가. ADR-025 §D5 Risk-2 mitigation 의 핵심 mechanism.
- **edge-case fixture deterministic seed** — 동일 seed → 동일 ticks 보장. retro audit (다음 PMO 회고) 재현 필수.
- **BacktestRunner callable injection** — `StrategyReproducibilityReporter` 가 `BacktestExecutor` 직접 의존하지 않음. production 은 executor wrapper, test 는 deterministic stub 를 주입하는 dual-use 패턴.
- **SignalProvenanceLog cross-reference** — Story-9 JSONL 의 signal_id ↔ snapshot trace 의 join 으로 dual-write 기간 어떤 bar 가 signal 을 발사했는지 byte-level 검증.
