## §4 Invariant (5 개)

- **INV-1**: Source WAL immutability — backfill 중 frozen WAL 무변경 (D3=A point-in-time snapshot)
- **INV-2**: Idempotency — 동일 backfill 반복 실행해도 final state 동일 (D4=A `.compacted` rename)
- **INV-3**: Schema 정합 — backfill L1 schema == MCT-166 path B L1 schema (`_ob_snapshot_dicts_to_arrow()` 재사용)
- **INV-4**: Partial boundary 박제 — frontmatter manifest 가 frozen 시작 ~ MCT-166 LAND 까지 date range 명시 (D5=B)
- **INV-5**: Verify gate — MCT-165 V2=0 AND 별 verify partial loss within threshold 양쪽 통과 후에만 §11 RETRO (D8=C)

## §5 Acceptance Criteria (7 개)

- **AC-1 (entry scan)**: Given MCT-166 fix LAND 직후, When Phase 2.1 entry scan 실행, Then frozen WAL path + freeze flag + pre-existing L1 inventory + partial WAL date boundary audit 박제
- **AC-2 (`--backfill` mode)**: Given frozen WAL segment list 입력, When `compactor.runner --backfill --exchange upbit --tier L1 --channel orderbooksnapshot` 실행, Then point-in-time snapshot 출력 + frontmatter manifest partial boundary 박제
- **AC-3 (idempotency)**: Given 동일 backfill job 재실행, When 기존 `.compacted` sentinel 존재, Then 해당 segment skip + source WAL 무변경
- **AC-4 (backfill 실행)**: Given Phase 2.3 trigger, When upbit orderbooksnapshot frozen WAL 전체 처리, Then L1 parquet row count > 0 + `_ob_snapshot_dicts_to_arrow()` schema 정합
- **AC-5 (MCT-165 verify)**: Given Phase 2.4 verify, When `data_health_check --tier L1 --exchange upbit --date <range>` 실행, Then V2 (forward-only loss) = 0
- **AC-6 (별 verify partial loss)**: Given Phase 2.4 별 verify, When frozen WAL row count vs L1 row count 비교, Then partial loss ratio 박제, threshold 초과 시 §10 FIX trigger
- **AC-7 (hub 박제)**: Given Phase 2 완료, When hub PR merge, Then Story §8-§12 + channel matrix MCT-166 result 아래 MCT-173 backfill result + RETRO
