- **신규 callsite (compactor/runner.py 통합)** = §5.4 Non-goal. 본 Story = `dual_write_window_runner.py` 만, 실 compactor callsite 통합은 별 Story 또는 본 Story 의 Phase 2 final integration 시점 (ArchitectPL §6.4 결정 — 분리).

**ADR-017 hot path 무영향 invariant 보존**:

- cron schedule (daily 03:00 KST) = 시간축 직교 (collector tick 과 직교)
- CompactionBarrier signal-based polling (MCT-151 land) 만 사용 — collector WAL/L1 ParquetWriter 침범 0
- IOPS 측정 = Prometheus query (read-only) — host process 영향 0

class DualWriteWindowRunner:
    """dual-write window 운영 layer — cron-style 매일 invariant verify + IOPS during 측정 + SOP 실전 가동.

    Responsibilities:
    1. cron orchestration (daily 03:00 KST default, configurable timezone + interval)
    2. CompactionBarrier.drain_and_block() invoke (cycle 진입 gate)
    3. InvariantHarness.verify() partition × symbol batch invoke (Phase B)
    4. drift detection + alert emission (Phase C — NASInvariantSchemaDriftDetected consume)
    5. IOPS during 측정 + ±15% gate verify (Phase D)
    6. SOPRunner.is_manual_gate() check (Phase A guard)
    7. evidence pack append (per-day report 누적)

    §8.5 active (process restart-aware):
    - file lock (POSIX flock or sqlite-based) 가 process restart 후에도 lock 잔존 → overlap 차단 유지
    - evidence pack file = append-only (process restart 후에도 partial write 0 보장 — fsync)
    - SOPRunner / CompactionBarrier signal 모두 file system 영속 (MCT-150/151 land 정합)

    ADR-017 hot path 무영향 invariant:
    - cron schedule = 시간축 직교 (collector tick 과 직교)
    - CompactionBarrier (MCT-151 land) 만 사용 — collector WAL/L1 ParquetWriter 침범 0
    - IOPS 측정 = Prometheus query read-only — host process 영향 0

    ADR-009 forward-only invariant 정합:
    - 본 runner = read-only verify layer (write 0)
    - 양쪽 storage 변경 0 (InvariantHarness.verify() 자체가 read-only invariant)

    §6.7 Cross-module contract (lesson #2 invariant):
    - DualWriteWindowResult.status switch 의무 (caller cron daemon)
    - InvariantResult / BarrierResult / SOPState propagate (재정의 0, MCT-150/151 SSOT)

    §6.9 placement:
    - Phase A/B/D/E = unconditional sequential (cycle 진입 후 무조건 실행)
    - Phase C = conditional (Phase B 결과 verify 후만 분기)
    """

    def __init__(
        self,
        invariant_harness: InvariantHarness,
        compaction_barrier: CompactionBarrier,
        sop_runner: NASUnreachableSOPRunner,
        metrics: PrometheusExporter,
        *,
        local_root: Path,
        nas_partition_root: str,
        partition_list_provider: callable,  # () -> list[tuple[str, str]] — (partition, symbol) tuple 반환
        evidence_pack_path: Path,
        lock_path: Path = Path("/data/dual_write_window.lock"),
        iops_query_url: str = "http://prometheus:9090",
        iops_15pct_baseline_p99_ms: float = 2870.65,  # MCT-148 T2 baseline 50MB p99
    ) -> None:
        self._invariant_harness = invariant_harness
        self._compaction_barrier = compaction_barrier
        self._sop_runner = sop_runner
        self._metrics = metrics
        self._local_root = local_root
        self._nas_partition_root = nas_partition_root
        self._partition_list_provider = partition_list_provider
        self._evidence_pack_path = evidence_pack_path
        self._lock_path = lock_path
        self._iops_collector = IOPSCollector(
            prometheus_url=iops_query_url,
            baseline_p99_ms=iops_15pct_baseline_p99_ms,
        )

    def run(self) -> DualWriteWindowResult:
        """Single cycle execution — Phase A → B → C → D → E sequential.

        Algorithm:
        Phase A (cycle entry — unconditional, §6.9):
          1. file lock 획득 (POSIX flock, EC-1 overlap 차단)
             - lock 획득 실패 시 → 즉시 return DualWriteWindowResult(status="barrier_drain_timeout",
               barrier_result=None, ...) + alert (overlap detection)
          2. self._sop_runner.is_manual_gate() check (EC-5):
             - True → release lock + return DualWriteWindowResult(status="sop_manual_gate", ...)
          3. self._compaction_barrier.drain_and_block() invoke:
             - status="ok"               → Phase B 진입
             - status="drain_timeout"    → release lock + return status="barrier_drain_timeout"
             - status="barrier_violated" → release lock + return status="barrier_drain_timeout" +
                                           emergency rollback signal emit

        Phase B (verify — unconditional sequential, §6.9):
          4. partitions = self._partition_list_provider()  # [(partition, symbol), ...]
          5. per_partition_results = {}
          6. for (partition, symbol) in partitions:
               result = self._invariant_harness.verify(
                 local_partition=self._local_root / partition,
                 nas_partition=f"{self._nas_partition_root}/{partition}",
               )
               per_partition_results[f"{partition}|{symbol}"] = result
               self._metrics.emit_invariant_verify(result.status, ...)

        Phase C (drift 판정 — conditional on verify result, §6.9):
          7. has_drift = any(r.status != "all_pass" for r in per_partition_results.values())
          8. if has_drift:
               status = "drift_detected"
               # NASInvariantSchemaDriftDetected alert rule (MCT-151 land) auto-fire from
               # nas_invariant_status_count metric — 본 runner emit 만, alert fire 는 Prometheus
             else:
               status = "healthy"

        Phase D (IOPS during 측정 — unconditional periodic, §6.9):
          9. iops_delta = self._iops_collector.snapshot(time_range_h=24)
         10. if not iops_delta.within_15pct_gate:
               status = "iops_gate_breached"  # override drift_detected/healthy (가장 우선 surface)
                                              # — but drift_detected 가 cutover 차단 더 critical 시
                                              # caller 에서 priority 조정 가능 (현 status priority:
                                              # sop_manual_gate > barrier_drain_timeout >
                                              # drift_detected > iops_gate_breached > healthy)

        Phase E (cycle exit — unconditional cleanup, §6.9):
         11. self._compaction_barrier.release()
         12. evidence pack append (per-day report)
         13. release file lock
         14. emit DualWriteWindowResult metric (status counter)
         15. return DualWriteWindowResult(status=..., ...)

        Returns:
            DualWriteWindowResult — caller (cron daemon) 가 status switch 의무.

        Raises:
            None — 모든 failure case 가 status enum 으로 propagate (caller switch 책임).
            단, 내부 IO error (file lock acquire 자체 실패) 는 OSError raise — cron daemon
            level retry policy (systemd OnFailure 등) 가 handle.

        Idempotency:
        - file lock 가 overlap 차단 (EC-1)
        - InvariantHarness.verify() 가 read-only — 다중 호출 시 동일 결과
        - CompactionBarrier.release() 가 idempotent (signal 부재 시 NO-OP)
        - evidence pack append 만 — file lock 보호 하 atomic write

        Metric emission (PrometheusExporter 경유 — 신규 prefix `nas_dual_write_window_*`):
        - nas_dual_write_window_status_count (Counter, labels: status [5 enum])
        - nas_dual_write_window_cycle_duration_seconds (Histogram, buckets [60, 300, 1800, 3600, 7200])
        - nas_dual_write_window_iops_delta_pct (Gauge, labels: metric_type [p99|read_iops|write_iops])
        - nas_dual_write_window_sop_trigger_count (Counter, labels: sop_state [3 enum])
        """

#### §7.4.1 DR (Disaster Recovery) — 운영 layer 측면

#### §7.4.2 Disconnect (NAS endpoint disconnect)

**위험**: cron cycle 중 NAS endpoint disconnect → InvariantHarness.verify() 의 NAS object download 실패 → false positive drift 검출.

**mitigation**:

#### §7.4.3 Clock (시간 기반 guard 의 system clock drift)
