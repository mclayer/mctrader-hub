### 변경 내용

**Transaction WAL** (`mctrader_data/wal/transaction_wal.py`):
- at-least-once + batch fsync 정책
- batch fsync interval: 10-100ms 또는 1000 msg (먼저 도달)
- WAL memory buffer: 50,000 msg (5000 msg/sec × 10sec burst 수용)
- SLA: power-loss window ≤ 100ms or ≤ 1000 msg

**Fallback tuple dedup** (`mctrader_data/wal/dedup.py`):
- Bithumb 미제공 unique trade_id → fallback tuple dedup
- Dedup key: `(exchange, symbol, ts_utc, price, quantity, side, raw_json_hash, ingest_seq)` (ADR-009 D10.7 reference)
- multi-node union 후 dedup (HA active-active 시)
- content mismatch → quarantine emit

- Compactor archive 실패 시 WAL 보존 24h → 7d 연장 (Story-7 협조)

### Lesson learned

- **Additive API pattern** — 기존 `tick_logical_key` 6-tuple / `deduplicate_ticks` 보존, 신규 `tick_v1_1_logical_key` 8-tuple / `deduplicate_ticks_v1_1` 별도. Story-4 의 ingest_seq/payload_hash 추가 column 으로 dedup precision 향상.
- **kwargs default = None pattern** — `fsync_window_ms` / `fsync_window_msgs` / `buffer_max_msgs` default None → legacy behavior. transaction tier 만 production tuning (별도 Story).
- **§8 Test Contract power-loss fixture (kill -9)** — 2 slow tests 가 SLA ≤ 1000 msg / ≤ 100 ms wall-clock 검증. Risk 1 (CRITICAL) mitigation 의 production-ready 박제.
