### 9.7 D13 cross-container mutex evidence (Task 14, 2026-05-08)

Linux Docker container (`mctrader-engine:mct-100`) 안에서 fcntl.flock cross-fd mutex 검증:

```
[D13 evidence] A acquired flock on /tmp/mct100-d13-evidence.lock
[D13 evidence] PASS: B got BlockingIOError (mutex enforced)
[D13 evidence] A released
[D13 evidence] PASS: B acquired after A released
[D13 evidence] cleanup OK
```

검증 의의: 두 distinct OFD (Open File Description) 가 동일 inode 위에서 `fcntl.flock(LOCK_EX|LOCK_NB)` 호출 시 두 번째는 `BlockingIOError` raise — kernel-managed mutex 가 cross-PID-namespace (즉, cross-container, 동일 named volume mount) 에서 정상 동작 검증. ADR-015 mutex invariant Docker 환경 보장.
