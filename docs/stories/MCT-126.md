# MCT-126: mctrader-web Main Dashboard + Nav 링크 + Smoke 테스트

- **Status**: 완료 (mctrader-web main 직접 push, CI green — 2026-05-10)
- **Trigger**: mctrader-web 사이드바 진입 시 별도 랜딩 페이지 부재 — 사용자 탐색 첫 화면이 status sidebar block 한 줄. Main 대시보드 + Nav 진입점 추가
- **Repos**: `mctrader-web` (단일 repo)

## 1. 배경

mctrader-web 멀티페이지 Streamlit 앱에 메인 진입점이 없었음. 사이드바의 그룹 expander 들 중 어느 것이 첫 화면인지 불분명하고, 전체 시스템 상태를 한눈에 볼 수 있는 대시보드 부재.

**Row 2**: Active Runs list (with strategy + status) + Engine Status grid (UP/DOWN/IDLE for 5 engines)

**Row 3**: Strategy Sets dataframe (latest 5) + Collector node summary

## 4. Pre-existing Bug Fix: idempotency cache 격리

**버그**: `tests/api/test_admin_signal_control.py` 전체 테스트가 동일 Idempotency-Key `"12345678-1234-4abc-89ab-123456789012"` 를 공유 → SQLite `idempotency_cache` DB가 테스트 실행 간 공유 → 재실행 시 캐시 히트가 docker verb 호출을 bypass

**수정**: `isolated_paths` 픽스처에 `MCTRADER_ADMIN_AUDIT_PATH` → `tmp_path` 리디렉션 + `reset_conn_for_testing()` 추가. 각 테스트가 독립 SQLite DB를 사용.
