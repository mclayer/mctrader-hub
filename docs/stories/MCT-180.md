## §6 Risk 상세

### R-MCT-180-1 integration smoke 10분 budget 초과 (MEDIUM)

- **위협**: compose up full stack + 60s ingest + compactor promotion + health check 합산 > 10분
  → CI timeout (GitHub Actions `timeout-minutes: 12`)
- **mitigation**:
  - infra (postgres+redis+minio+mc) `--wait --wait-timeout 180` 선행
  - collector + paper-engine `--wait --wait-timeout 240` 별도 단계
  - SIGTERM step 이전 health 200 확인 (불필요한 대기 차단)
  - 실패 시 teardown `if: always()` 보장 (CI cost 낭비 방지)

### R-MCT-180-2 testcontainers CI 환경 호환성 (LOW)

- **위협**: ubuntu-latest GitHub Actions 환경에서 testcontainers-python Docker socket 접근 미허용
  → CI FAIL
- **mitigation**:
  - `testcontainers-python` = GitHub Actions ubuntu-latest Docker socket 기본 허용
  - MinIO testcontainer = `minio/minio:latest` (공개 image, ghcr.io 인증 불필요)
  - Redis testcontainer = `redis:7-alpine` (공개 image)

### R-MCT-180-3 resource limits OOM kill (MEDIUM)

- **위협**: 실 production 부하 시 reader cache 256MB + DualWriter buffer 초과 → OOM kill
- **mitigation**:
  - paper-engine limits 512M: reader_cache 256MB (MCT-170 max_bytes default) + asyncio runtime + 여유 버퍼
  - collector limits 512M: INV-4 DualWriter peak RSS+TM delta ≤ 50MB (MCT-163 실측) + 여유
  - cadvisor ContainerMemoryHigh (>80% = 410M) → 선제 alert + D11 amendment 발의 경로 보유
