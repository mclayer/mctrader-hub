## §6 Risk 상세

### R-MCT-179-1 WAL 측정값 production 괴리 (HIGH)

- **위협**: paper-synthetic 측정값 이 production 실 peak (50sym × 3ch × 12seg/h burst) 와 크게 다를 경우
  → D11 amendment 발의 판단 오류. 15~45G ±50% hypothesis 벗어날 가능성.
- **mitigation**:
  - synthetic = paper mode (MCT-172 D8-2 패턴) 30min baseline + 30min peak burst hybrid
  - 초과 시 즉시 exit 7 + GitHub issue 자동 발의 → 사용자 판단 (amendment or defer)
  - production 실 측정 = 별 PR carry over (실 production deploy 후, peak 09:00 KST 1h window)
- **잔여 risk**: production 측정 전까지 D11 30G 기준 확신 불가. production 별 PR 의무 유지.

### R-MCT-179-2 Prometheus scrape fail (MEDIUM)

- **위협**: collector / paper-engine 의 /metrics endpoint 미노출 또는 포트 불일치 → scrape miss
  → alert rule 미발화 (silent WAL 초과)
- **mitigation**:
  - compose.yml 에서 9090 포트 명시 expose + prometheus.yml scrape job 정합 verify (AC-3)
  - Prometheus scrape target state = up 확인 의무 (AC-3)
  - paper-engine 기존 HealthServer :8080 + /metrics 별 endpoint 분리 (MCT-177 LAND asyncio SSOT)

### R-MCT-179-3 InvariantHarness startup scan overhead (LOW)

- **위협**: 8종 scan = I/O 증가 → container cold start 지연 (기존 60s SIGTERM grace 압박)
- **mitigation**:
  - warn+continue (hard abort 아님) — scan 실패 시 log only + 정상 기동 계속 (ADR-030 §D17)
  - MCT-180 testcontainers smoke 에서 startup time 회귀 검증
