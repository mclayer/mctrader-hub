### 도메인 용어 정합

- **cold reader endpoint flip**: mctrader-engine 의 cold L2/L3 read source 를 local volume → NAS endpoint 로 single config swap (`MINIO_ENDPOINT` env 갱신). atomicity = 동시 진행 read 의 partial state 노출 0 (mutex or immutable swap). hot tier (L1) reader = scope 외 (local volume only 유지).
- **reader cache (read-through cache, ADR-027 D9 박제)**: mctrader-engine 호스트 측 LRU + TTL cache layer. NAS GET latency (MCT-148 T2 p99 측정값 base) 의 hot read amortize. local cache = read latency optimization, write authority 0 (NAS = SoT).
- **cache flush + verify barrier (S3 박제)**: endpoint flip 전 cache 의 모든 entry 강제 무효화 + flush 완료 verify gate. flush 완료 verify = (a) cache size == 0 (b) verify probe (test entry put → flush → get == None). flush partial fail 시 즉시 alert + endpoint flip 차단.
- **dual-write 7d grace 연장 (S9 박제)**: cutover 직후 dual-write window (MCT-152 dual_write_window_runner) 를 7일 추가 운영. cutover 후 NAS 측 silent 문제 발견 시 7d grace 안에 endpoint rollback 가능 (local volume 측 데이터 살아 있음 = belt-and-suspenders).

- **engine smoke test (cutover 직후 검증)**: sample cold L2 partition 5+개 NAS endpoint read 검증. (a) read latency p99 MCT-148 T2 baseline ±15% gate (b) read 결과 byte-level identity sha256 비교 (c) 16-col schema invariant 4종 ALL PASS (d) read-through cache hit ratio 정상 동작 (e) legacy `node=` 부재 partition 1+개 포함 (AC-4 cross-reference).
- **endpoint flip atomicity**: flip 진행 중 동시 진행 read 의 partial state 노출 0. mutex (read 진행 중 flip block) 또는 immutable swap (config object 단위 atomic replace) 또는 read-write lock (cutover 시 short window 만 read block) — ArchitectPL §6 결정.

### §5.1 Acceptance Criteria (Issue #270 body 박제, Given/When/Then 확장)

**AC-1 (cold_reader endpoint atomic flip — local → NAS, single config swap)**

- **Given** mctrader-engine 의 cold L2/L3 reader 가 local volume 기반 read 운영 중 + `MINIO_ENDPOINT` env 가 local volume placeholder (또는 미설정) 상태 + endpoint_router + cold_reader 모듈 신규 land + dual_write_window_runner 가 7d grace mode 활성 (또는 cutover 직후 활성화 의도)

- **Then**
  - endpoint_router 가 env reload + endpoint resolution
  - flip atomicity guarantee — 동시 진행 read 의 partial state 노출 0 (mutex / immutable swap / read-write lock, ArchitectPL §6 결정)
  - flip 진행 중 신규 read 는 cache flush 완료 후 (AC-2 cross-reference) NAS endpoint 진입
  - flip 시점 evidence emit (Prometheus metric `engine_cold_reader_endpoint_flip_total` + log entry, ArchitectPL §6 metric naming 결정)

**AC-2 (reader cache explicit flush + verify barrier — S3 박제, endpoint flip 전 stale cache 차단)**

### §5.2 NFR (codeforge develop / test convention 준수)

- **NFR-1 (cutover atomicity 의무, S3+AC-1+AC-2 enforcement)**: endpoint flip + cache flush + verify barrier 전 과정의 atomicity 의무. flip 진행 중 동시 진행 read 의 partial state 노출 0 (AC-1 mutex/immutable swap). cache flush 진행 중 동시 진행 read 의 stale entry hit 0 (AC-2 mutex/copy-on-write). flush partial fail 시 endpoint flip 차단 (AC-2 verify gate). ArchitectPL §6.9 unconditional placement 의무.

### §5.3 Edge Cases

- **EC-1 (cache flush partial fail — verify probe FAIL)**: AC-2 cache flush 진행 중 일부 entry 만 무효화 + 나머지 entry 잔존 → verify probe (test entry put → flush → get == None) FAIL → 즉시 alert + endpoint flip 차단 + retry mechanism (ArchitectPL §6 결정, 권고: 3 retry budget). retry 모두 FAIL 시 user manual gate (cutover 차단 + cache fresh process restart 권고). **사용자 directive verbatim "절대 유실 금지" 의 cache 측 fail-safe** — flush partial fail 시 cutover 차단이 stale cache hit 보다 우선.
- **EC-2 (dual-write 7d 연장 중 NAS unreachable)**: AC-3 7d grace 운영 중 NAS endpoint 단절 발생 → mctrader-engine cold_reader read fail (NAS unreachable) + mctrader-data writer 측 RetryQueue enqueue (MCT-150 RetryQueue, NASUploader.put() PutResult.status=`hard_floor_blocked` 가능). engine 측 graceful degradation mechanism 결정 의무 (ArchitectPL §6) — cache fallback (cache 의 stale entry serve, but cache flush 후이므로 hit 0) vs alert + read 차단 vs MCT-152 SOPRunner state machine 통합 (MANUAL_GATE 진입 시 read 차단). ReadResult.status=`nas_unreachable` enum return + alert escalation. **7d grace 중 NAS 단절 = endpoint rollback trigger** (사용자 결정, runbook 박제).
- **EC-3 (legacy partition discovery during cutover — read miss)**: AC-4 cold_reader 의 legacy `node=` 부재 partition read mapping 정합 — but MCT-153 backfill 측에서 PUT 누락 partition (예: backfill orchestrator 가 closed-day filter 로 일부 partition skip) 발견 시 cold_reader read miss (404) → ReadResult.status=`not_found` enum return + alert + 사용자 manual gate. **MCT-153 backfill metric ↔ engine reader metric 1:1 정합 검증 (AC-4 cross-reference)** 가 사전 차단 mechanism — backfill metric 누적 count 와 reader metric 누적 count 가 partition 단위 일치 의무 (drift 검출 시 cutover 차단).
- **EC-4 (engine smoke FAIL on first read)**: AC-5 cutover 직후 첫 smoke test 가 (a) latency p99 gate 위반 (MCT-148 T2 ±15% 초과) (b) sha256 mismatch (c) 16-col schema invariant FAIL (d) cache hit ratio 0 (e) legacy partition read miss 중 1종 이상 FAIL → 즉시 endpoint rollback (dual-write 7d grace 활용, AC-3 cross-reference) + alert escalation. rollback mechanism = endpoint_router.rollback() 호출 (env 원복 + endpoint resolution 재진입) + cache flush 재실행 + sample partition 재read smoke. 재시도 mechanism 박제 (ArchitectPL §6 결정, 권고: 1 retry + 사용자 manual gate). rollback 시점 evidence 박제 (Prometheus metric `engine_cold_reader_rollback_total` + log).
- **EC-5 (cutover rollback — endpoint 원복 후 재시도)**: EC-2/EC-4 trigger 시 endpoint_router.rollback() 호출 → `MINIO_ENDPOINT` env 원복 (NAS → local volume placeholder 또는 미설정) + endpoint resolution 재진입 + cache flush 재실행 + sample partition 재read smoke. 재시도 횟수 = ArchitectPL §6 결정 (권고: 1 retry, fail 시 user manual gate). dual-write 7d grace 만료 시점 reset (rollback 발생 시 grace counter reset) — ArchitectPL §6 결정.
- **EC-6 (cache TTL expiry vs explicit flush ordering race)**: AC-2 cache flush 호출 직전 TTL expiry mechanism 가 background thread 에서 동시 발생 → 일부 entry 가 TTL expiry path + 나머지 entry 가 explicit flush path 로 무효화 → mixed state. flush + verify barrier 가 양쪽 path 결과 모두 무효화 검증 의무 (ArchitectPL §6 결정 — TTL thread 와 explicit flush 의 mutex 또는 ordering guarantee).

**chief decision 6건**:

1. **endpoint flip atomicity mechanism** = **immutable swap** 채택 (mutex / read-write lock 거부). 근거:
   - Python GIL = 단순 attribute 재할당 (`self._endpoint_url = new_url`) 자체가 atomic (CPython bytecode 단일 STORE)
   - boto3 client = thread-safe (botocore documentation 박제), client 재생성 시 신규 client 즉시 사용 가능
   - mutex 채택 시 read 진행 중 flip block → cutover window 동안 read latency degradation
   - read-write lock 채택 시 reader 측 lock acquire 비용 증가 + Python `threading.Lock` 의 read-write lock 부재 (별 구현 의무, 비용 정당화 0)
   - **immutable swap = `endpoint_router._client = boto3.client("s3", endpoint_url=new_url)`** + `self._endpoint_url = new_url` (bytecode 단일 STORE) → 동시 진행 read 의 partial state 노출 0 (이전 client 사용 중인 read 는 in-flight 완료, 신규 read 는 신규 client 사용)

#### §6.2.2 `ReaderCache` class (`src/mctrader_engine/io/reader_cache.py`)

```python
"""LRU + TTL cache (ADR-027 D9 박제) + explicit flush API + verify barrier (S3 박제).

Responsibilities:
1. LRU eviction (capacity bound) + TTL expiry (time-based eviction) — read-through cache
2. explicit flush API (`flush_all()`) — endpoint flip 전 cache 강제 무효화 (S3, AC-2)
3. verify barrier (`verify_flushed()`) — flush 완료 verify gate (AC-2 enforcement)
4. cache hit/miss metric emit — AC-5 smoke test 측 hit ratio 측정

ADR-027 D9 직접 owner (read-through cache, "MCT-154 scope" 박제).
S3 박제 직접 enforcement — endpoint flip 전 stale cache 차단.

§8.5 active (background worker = TTL eviction thread):
- TTL eviction = background thread (periodic sweep 또는 lazy eviction on access)
- restart 후 cache cold start (in-memory only, persistence 0) — 정상 동작 (cache miss -> NAS read)

§6.7 Cross-module contract (lesson #2 invariant):
- CacheFlushResult.status switch 의무 (caller endpoint_router.flip())
- ReadResult enum의 hit_cache 결과 propagate (caller cold_reader.read())

§6.9 placement:
- get/put = unconditional (매 read 시 호출, idempotent)
- flush_all = unconditional placement (endpoint flip 진입 직전 첫 단계, S3 enforcement, AC-2)
- verify_flushed = unconditional placement (flush_all 후 즉시 호출, AC-2 verify gate)
- TTL eviction = background thread conditional (TTL expiry 시점 only)
"""
from __future__ import annotations

from collections import OrderedDict
from dataclasses import dataclass
from typing import Literal, Optional
import threading
import time
import uuid


@dataclass(frozen=True)
class CacheFlushResult:
    """cache flush 결과 — caller switch 의무 (§6.7 cross-module contract).

    status enum (3종, §6.8.1 SSOT):
    - "flushed"               : 모든 entry 무효화 + verify probe 정상 통과 (AC-2 정상 case)
    - "partial_flush_failed"  : flush 진행 중 일부 entry 만 무효화 (race or internal error)
    - "verify_probe_failed"   : flush 완료 후 verify probe (test entry put -> flush -> get == None) FAIL

    Caller 처리 의무 (§6.7 매핑):
    - "flushed"               -> endpoint_router.flip() 정상 진행 (Phase 2)
    - "partial_flush_failed"  -> retry (budget §6.1 결정 = 3회) + retry 모두 fail 시 alert + cutover 차단
    - "verify_probe_failed"   -> alert (`EngineColdReaderCacheFlushVerifyFailed`) + cutover 차단 + 사용자 manual gate
    """

    flushed_count: int = 0           # flushed entry count (verify input)
    remaining_count: int = 0         # partial_flush_failed 시 잔존 entry count
    verify_probe_key: str = ""       # verify_probe_failed 시 박제 (debug input)
    flush_duration_ms: float = 0.0


@dataclass
class CacheEntry:
    """cache entry — value + insert timestamp (TTL expiry 계산).

    Fields:
    - value           : cached bytes (parquet partition data)
    - inserted_at     : monotonic timestamp (TTL expiry 계산용)
    - last_accessed_at: monotonic timestamp (LRU eviction 계산용)
    """
    value: bytes
    inserted_at: float
    last_accessed_at: float


class ReaderCache:
    """LRU + TTL cache + explicit flush + verify barrier.

    Thread-safety:
    - mutex-protected dict swap (§6.1 chief decision 2) — flush 진행 중 read 의 stale entry hit 0
    - get/put 도 lock 보호 (LRU OrderedDict.move_to_end + 신규 insert 시 capacity check)

    Cache eviction policy:
    - LRU: capacity 도달 시 가장 오래 미접근 entry evict
    - TTL: inserted_at + ttl_seconds < now 시 entry evict (lazy on access + periodic background sweep)

    Verify probe (AC-2 enforcement):
    - dedicated key namespace `__verify_probe_<uuid>` 사용
    - flush_all() 후 verify probe key put -> flush 재실행 -> get == None verify
    - LRU/TTL evict mechanism 와 race 0 (probe key 가 다른 read 와 충돌 0)
    """

    def __init__(
        self,
        *,
        capacity: int = 1024,        # max entry count (LRU bound, §6.1 chief decision)
        ttl_seconds: float = 300.0,  # 5min TTL (NAS GET latency hot read amortize, ADR-027 D9 박제)
    ) -> None:
        self._capacity = capacity
        self._ttl_seconds = ttl_seconds
        self._cache: OrderedDict[str, CacheEntry] = OrderedDict()
        self._lock = threading.Lock()
        self._hit_count = 0
        self._miss_count = 0

    def get(self, key: str) -> Optional[bytes]:
        """cache lookup — LRU update + TTL check + hit/miss metric emit.

        Algorithm:
        1. with self._lock:
           a. entry = self._cache.get(key)
           b. if entry is None -> self._miss_count += 1; return None
           c. if entry.inserted_at + self._ttl_seconds < time.monotonic() -> TTL expired, evict; self._miss_count += 1; return None
           d. entry.last_accessed_at = time.monotonic()
           e. self._cache.move_to_end(key)  # LRU update
           f. self._hit_count += 1
           g. return entry.value

        Idempotency: 다중 호출 시 동일 결과 (read-only operation + LRU/TTL deterministic).
        """
        ...  # Phase 2 impl

    def put(self, key: str, value: bytes) -> None:
        """cache populate — LRU capacity check + 신규 insert.

        Algorithm:
        1. with self._lock:
           a. now = time.monotonic()
           b. if key in self._cache -> 기존 entry 갱신 (value + inserted_at + last_accessed_at)
           c. else:
              - if len(self._cache) >= self._capacity -> LRU evict (popitem(last=False))
              - self._cache[key] = CacheEntry(value, now, now)
           d. self._cache.move_to_end(key)  # 신규/갱신 모두 LRU 정합

        Idempotency: 다중 호출 시 결과 동일 (마지막 put 만 보존, value 동일 시 timestamp 만 갱신).
        """
        ...  # Phase 2 impl

    def flush_all(self) -> CacheFlushResult:
        """모든 entry 강제 무효화 + verify probe (AC-2 enforcement, S3 박제 핵심).

        Algorithm (mutex-protected dict swap pattern, §6.1 chief decision 2):
        Phase 1 (flush, unconditional, §6.9):
        1. start = time.monotonic()
        2. with self._lock:
           a. flushed_count = len(self._cache)
           b. self._cache = OrderedDict()  # 신규 OrderedDict 할당 (immutable replace)
           c. self._hit_count = 0; self._miss_count = 0  # metric reset

        Phase 2 (verify probe, unconditional, §6.9):
        3. probe_key = f"__verify_probe_{uuid.uuid4().hex}"
        4. probe_value = b"verify_probe"
        5. self.put(probe_key, probe_value)
        6. flush 재실행 (mutex-protected):
           with self._lock:
               self._cache.pop(probe_key, None)  # explicit removal (verify probe 만)
               # 다른 entry 가 race 로 추가된 경우 검출
               remaining = len(self._cache)
        7. probe verify:
           if self.get(probe_key) is not None -> "verify_probe_failed" return
           if remaining > 0 -> "partial_flush_failed" return (race detection)
        8. flush_duration_ms = (time.monotonic() - start) * 1000
        9. metric emit (engine_cold_reader_cache_flush_total{status="flushed"} + engine_cold_reader_cache_flush_verify_result)
        10. return CacheFlushResult(status="flushed", flushed_count=flushed_count, ...)

        Idempotency (§11.6): 다중 호출 시 동일 결과 (cache 가 이미 empty 시도 정상 flush + verify probe 정상 pass).
        """
        ...  # Phase 2 impl

    def verify_flushed(self) -> bool:
        """flush 완료 verify gate (AC-2 enforcement) — caller endpoint_router.flip() 진입 직전 호출.

        Returns:
            bool — True (cache empty + verify probe pass) / False (잔존 entry 검출)

        Algorithm:
        1. with self._lock:
           a. if len(self._cache) > 0 -> return False
        2. probe_key = f"__verify_probe_check_{uuid.uuid4().hex}"
        3. self.put(probe_key, b"check")
        4. with self._lock:
           a. self._cache.pop(probe_key, None)
        5. result = self.get(probe_key) is None
        6. return result

        Idempotency: 다중 호출 시 동일 결과 (read-only verify, side effect 별).
        """
        ...  # Phase 2 impl

    def hit_ratio(self) -> float:
        """cache hit ratio return — AC-5 smoke test 측 측정값 (LRU+TTL 정상 동작 verify).

        Returns:
            float — hit_count / (hit_count + miss_count), 0.0 if no access yet
        """
        total = self._hit_count + self._miss_count
        return self._hit_count / total if total > 0 else 0.0

    def stats(self) -> dict:
        """cache 통계 return — caller (smoke test / metric emit) 측 활용.

        Returns:
            dict — size + capacity + ttl_seconds + hit_count + miss_count + hit_ratio

                "miss_count": self._miss_count,
                "hit_ratio": self.hit_ratio(),
            }
```

#### §6.2.3 `ColdReader` class (`src/mctrader_engine/io/cold_reader.py`)

### §6.12 Threading + concurrency 모델 (chief synthesis)

본 Story 3 module 의 threading model:

| module | thread context | concurrency primitive | safety mechanism |
|---|---|---|---|

### §7.4 OperationalRiskArchitect — 운영 리스크 5 항목 (CFP-46 mandatory)

#### §7.4.1 DR (Disaster Recovery) — cutover 운영 측면

**위험**: cutover 진행 중 (Phase 1~3, 추정 ~30-60분) host process kill / 컨테이너 restart / NAS endpoint 단절 / cutover_state.yaml 손상 → cutover 중단 + state 손실 → endpoint flip partial state (env 갱신 후 grace mode 활성화 전 process down) → 재시작 시 endpoint flip 재진입 + duplicate state.

**mitigation**:

- **endpoint_router.flip() idempotent** (§11.6) — 다중 호출 시 동일 결과 (atomic swap = 동일 값 STORE), 재진입 안전
- **cutover_state.yaml git commit audit trail** — git history 가 cutover 진행 timeline 박제 (PR + commit timestamp)
- **runbook 박제 manual sequential checklist** — operator 가 1번씩 확인 + 결과 박제 (Phase 0~4 + Rollback)
- **reader_cache restart cold start 정상 동작** — in-memory only, persistence 0 → restart 시 cache miss → NAS read 정상 fallback
- **endpoint_router restart grace state 정합 read** — cutover_state.yaml 측 persistent state load (§8.5 active 정합)

**잔존 리스크**: cutover_state.yaml 손상 시 (file system 측 corruption) grace mode state 손실 → 7d grace 진행 중 → cutover 진입 시점 reset (grace_started_at 갱신) 가능 → 재7일. RPO=0 위반 0 (NAS object 자체는 retain — cutover 측 reset 만, data 영향 0).

**복구 절차 (operator runbook 권고)**:
1. cutover_state.yaml backup (git history 측 직전 commit revert 가능)
2. `endpoint_router._reload_grace_state()` 강제 재호출 (process restart)
3. grace_remaining_days() 계산 결과 verify — 손실 시 재7일 reset 결정 (사용자 결정)

#### §7.4.2 Disconnect (NAS endpoint disconnect during cutover or 7d grace)

**위험**: cutover 진행 중 (Phase 1~3) NAS endpoint disconnect → smoke test (Phase 3) 측 NAS read 다수 fail → cutover 차단 (rollback 진입). 7d grace 진행 중 NAS endpoint disconnect → cold_reader read fail (NAS unreachable) + dual-write 측 NASUploader RetryQueue enqueue (MCT-150 SOP).

**mitigation**:

- cutover Phase 0 gate 측 NAS endpoint 가용성 verify (`mc admin info <alias>` 또는 `boto3 head_bucket` 등 사전 ping)
- cold_reader.read() = nas_unreachable graceful degradation (cache fallback 0 since cache 가 flush 후 cold start, alert + read 차단)
- 7d grace 중 NAS 단절 = endpoint rollback trigger (사용자 결정, runbook 박제 — `endpoint_router.rollback()` invoke + cache flush 재실행)
- MCT-152 NASUnreachableSOPRunner state machine consume — 7d grace 중 NAS 단절 시 SOPRunner MANUAL_GATE 진입 → cold_reader 측 graceful read 차단 (alert + 사용자 manual gate)
- AC-5 chaos test = cutover 직후 smoke test FAIL 시나리오 enforce — endpoint rollback path 직접 검증

**잔존 리스크**: 7d grace 만료 직전 NAS 단절 시 GC 진입 prerequisite 미충족 → grace 만료 reset 또는 재7일 결정 (사용자 결정, runbook 박제).

#### §7.4.3 Clock (system clock drift — 7d grace 만료 시점 판정)

**위험**: 7d grace 진행 중 system clock drift → grace_remaining_days() 계산 부정확 → grace 만료 시점 부정확 → MCT-155 GC 진입 시점 부정확.

**mitigation**:

- mctrader 호스트 = NTP 동기화 의무 (운영 baseline) — system clock drift sub-second 가정
- grace_remaining_days() = `(grace_started_at + grace_days - now).days` 계산 — sub-day precision 부정확 영향 minimal (만료 ±1일 여유)
- grace_started_at = ISO8601 timestamp 박제 → cutover_state.yaml git commit 시점 recorded (audit trail 정합)
- cutover Phase 4 7d grace 활성화 시점 evidence 박제 — Prometheus metric `engine_dual_write_grace_remaining_days` countdown 가 별 source (NTP 동기화 가정)
- **NTP drift detection**: `endpoint_router._reload_grace_state()` 내 wall-clock vs monotonic delta 측정 → drift > 5min 시 alert 후보 (Phase 2 impl 시점 박제 권고 deferred)

**잔존 리스크**: NTP 부재 환경에서 grace 만료 부정확. mctrader 운영 baseline = NTP 동기화 의무 — 본 Story scope 외, but operator runbook 박제 권고 (NTP 동기화 verify, Phase 0 gate).

#### §7.4.4 Rate-limit (NAS GET rate-limit + reader_cache 효과)

**위험**: cutover 직후 cold L2/L3 reader 가 NAS endpoint 측 GET 다수 진행 → MinIO server-side throttle 가능성 (MinIO RELEASE.2025-04-08 의 rate-limit default = 무제한 but 실 측정 부재) + reader_cache 효과 미확인.

**mitigation**:

- reader_cache LRU+TTL = NAS GET latency hot read amortize (ADR-027 D9 박제) — cache hit ratio > 0 시 NAS GET 횟수 감소
- cutover 직후 cache cold start (flush 후) → 첫 read 다수 NAS GET 진행 → 점진적 cache populate → 안정 상태 cache hit ratio 증가
- AC-5 smoke test = cache hit ratio 측정 의무 (LRU+TTL 정상 동작 verify)
- MCT-148 T2 latency baseline (50MB p99 = 2870.65ms) ±15% gate cross-reference — drift 검출 시 alert
- 7d grace 진행 중 read 패턴 monitoring — Prometheus metric `engine_cold_reader_read_latency_ms` p99 + cache hit ratio cross-reference

**잔존 리스크**: cutover 직후 cold start period (~수분~수시간 추정) 동안 NAS GET burst → MinIO throttle 가능성. evidence pack §5 Cache Hit Ratio 박제 → RETRO 시점 cold start period 측정 + Phase 2 impl 시점 reader_cache pre-warm 옵션 검토 권고 (deferred to MCT-155 또는 별 follow-up).

#### §7.4.5 Env-isolation (mctrader-engine 컨테이너 측 cutover trigger 환경 분리)

**위험**: cutover trigger (operator manual) 가 mctrader-engine 컨테이너 측 직접 실행 (`docker exec`) — env 분리 0 + secret leak surface 확장 + dependency 충돌 위험.

**mitigation**:

- **결정 후보 1 — mctrader-engine 컨테이너 내 endpoint_router.flip() 직접 호출** (사용자 manual `docker exec`): 기존 컨테이너 entrypoint 측 Python REPL 또는 신설 CLI 가용 + env 분리 0 — 단일 컨테이너 의무 정합 (운영 단순화)
- **결정 후보 2 — 별 cutover sidecar 컨테이너**: 분리 의무 강제 — but Stage 2 scope 비대화 + sidecar 통신 비용 + 단일 cutover 운영 (1회) 에 sidecar 정당화 0
- **ArchitectPL §7.4.5 chief decision = 후보 1 채택** (mctrader-engine 컨테이너 내 직접 실행) — 근거: (a) Stage 2 scope 단순화 + 1회 cutover 운영 (b) MCT-150~153 패턴 정합 (mctrader-data 측 CLI 도 컨테이너 내 실행) (c) env 분리 = `.env` 0600 + gitignored mitigation 충분 (MCT-147 4중 mitigation 정합) (d) 단일 cutover 1회 운영 = container restart 영향 0 (immutable swap idempotency + cutover_state.yaml persistent + reader_cache cold start 정상)
- **잔존 의무 박제**: Phase 2 impl 시점 cutover trigger procedure operator runbook 박제 (Phase 1 후속 = 본 §6.5 골격 → Phase 2 impl 시점 상세 박제) — `docker exec mctrader-engine python -c "from mctrader_engine.io import EndpointRouter; ..."` example 박제

**잔존 리스크**: 단일 컨테이너 내 cutover trigger 시 컨테이너 restart 시 cutover 중단 (state lost) — but immutable swap idempotency + cutover_state.yaml persistent + reader_cache cold start 정상 fallback 으로 재진입 안전 (file system 영속, §8.5 active 정합).

### §7.5 Threat boundary (SecurityArch — trust boundary)
