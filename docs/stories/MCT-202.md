### Edge Case 박제 (RequirementsAnalystAgent)

- **E-1: NAS 5xx retry_queue path race** — `result.status == 'local_only'` (queued) 분기 진입 직후 retry worker 가 즉시 retry → 성공 → `belated commit`. 이 경우 source 는 이미 local 보존 상태. **derived default**: belated commit 후에도 source 보존, 다음 sweep cycle 의 `scan_and_cleanup_legacy` 가 회수 (carrier 보존, 사용자 답변 #2 자연 정합).
- **E-2: Cross-tier 경쟁 (L1→L2 진행 중 L2→L3 trigger)** — `lockfile` 또는 `.compacted` sentinel 양쪽 모두 atomic guard 의무. **derived default**: 기존 `.compacted` sentinel 메커니즘 (ADR-017 §D2) 재사용 — L2 source parquet 이 L2→L3 cascade 중간에 unlink 되어도 L3 compactor 는 이미 L2 NAS object 를 GET (ADR-034 layout 정합) 하므로 functional 영향 0.

### Invariant (Architect lane 박제 의무)

- **INV-A**: eager unlink ↔ `scan_and_cleanup_legacy` sweep race → FileNotFoundError graceful no-op (`contextlib.suppress(FileNotFoundError)` 패턴).
- **INV-B**: pre-delete HEAD guard (`promote_l1` 의 기존 4중 HEAD verify — ETag + VersionId + sha256 Metadata + ContentLength) 재사용. caller 가 별 guard 추가 0.
- **INV-C**: forward-only 3-tier 확장 — ADR-009 §D12.2 annotation 의무 (본 Story §11 amendment 후보).
- **INV-D**: `result.status == 'committed'` XOR `source exists` — committed 이면 source unlink, 아니면 source 보존 (mutually exclusive).
