## §4 Invariant (5)

- **INV-1**: Hot path 무영향 (ADR-027 §D5 정합) — versioning + Object Lock 도입이 collector / L1 ParquetWriter latency 영향 0
- **INV-2**: DeleteMarker 30d 보존 (Object Lock governance + Lifecycle NoncurrentVersionExpiration 30d 정합)
- **INV-3**: Lifecycle ILM — storage cost 1.5x 미만 (append-only 1.1x-1.3x 예상)
- **INV-4**: DR runbook SSOT — restore-from-version 단계 단일 경로
- **INV-5**: replication 후속 별 Story (D2=D) — Phase 2 retro 시 reservation 의무

## §5 AC (5)

- **AC-1**: `put_bucket_versioning(Status: Enabled)` → `get_bucket_versioning()=Enabled` + delete 시 DeleteMarker 생성
- **AC-2**: Object Lock GOVERNANCE 30d 적용 → 30d 이내 version delete violation
- **AC-3**: Lifecycle NoncurrentVersionExpiration 30d → 30d 경과 version 자동 expiration
- **AC-4**: DR runbook 5-step (Triage / Version 조회 / Restore / Verify / Postmortem) 실행 가능
- **AC-5**: ADR-027 §D MCT-161 amendment LAND + history 추가

### Edge (2)
- **Edge-1**: 기존 1884 L1 parquet "null" version (versioning enable 전) — 신규 부터 v1+
- **Edge-2**: Object Lock + hot path latency 회귀 0 (Integration Test 5 검증)

## §6 Risk (3)

- **R1 Mid**: governance 30d + lifecycle 30d mismatch → DeleteMarker 보존 못 함. **완화**: 양쪽 30d 정합
- **R2 Low**: 기존 null version evidence 박제 불가. **완화**: Edge-1 명시, list_object_versions 신규 v1+ 검증
- **R3 Mid**: single NAS box → replication target 물리 부재 → D2=D 정당화. **완화**: ADR amendment + 후속 Story reservation
