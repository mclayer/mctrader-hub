## 2. Fix 브랜치 현황

### `fix/data-storage-atomic`
| 항목 | 내용 |
|---|---|
| `storage.py` | `_candles_to_arrow()`: 16컬럼 전체 emit — `trade_count`, `is_complete`, `source_ingested_at`, `data_snapshot_id`, `data_hash`(SHA-256) 추가 |
| `storage.py` | 원자 쓰기: `.tmp_{uuid}_{filename}` → `rename()` (POSIX/Windows 모두) |
| `storage.py` | 중복 방지: 기존 파일의 data_hash set과 비교 → 일치 시 skip |

### `fix/data-policy-wiring`
| 항목 | 내용 |
|---|---|
| `policy.py` | `candle_hash()`, `check_schema()`, `check_gap()`, `check_value_range()`, `check_duplicate()` 신규 구현 |
| `cli.py` | backfill loop에 6-policy 완전 연결: gap halt / dup same-hash skip / dup diff-hash quarantine / OOR quarantine / schema mismatch halt / API retry+halt |
| `cli.py` | quarantine: `{root}/quarantine/{exchange}_{sym}_{tf}_{ts}_{reason}.json` 기록 |
| `cli.py` | 결과 요약: `written=N skipped=N quarantined=N` |

### `fix/data-collector-reliability`
| 항목 | 내용 |
|---|---|
| `collector.py` | `asyncio.gather(return_exceptions=True)` + `_run_with_restart()` — 개별 symbol 5s/30s/120s backoff 재시작 |
| `dedup.py` | `compute_data_hash(candle)` SHA-256, `DedupStatus(Enum)`, `is_duplicate()` data_hash 기반 |
| `lineage.py` | `.tmp_{uuid}_{name}` → `rename()` 원자 쓰기 |
| `health_server.py` | `last_heartbeat_ts` staleness check → `MAX_STALE_SECONDS`(기본 60s) 초과 시 HTTP 503 |

## 3. Follow-up (이 Story 내 처리 필요)

### heartbeat.py: `last_heartbeat_ts` 설정 누락
`health_server.py`가 `heartbeat_writer.last_heartbeat_ts`를 읽지만,
`HeartbeatWriter.write_once()`에서 이 속성을 설정하지 않음 → health server가 항상 503 반환.

**수정**: `heartbeat.py`의 `write_once()` 내 성공 분기에
```python
self.last_heartbeat_ts = datetime.now(timezone.utc)
```
추가. (한 줄 수정)
