### 4 핵심 도메인 사실 (DomainAgent 박제)

1. **L1Compactor architecture (CodebaseMapperAgent fetch)**: `src/mctrader_data/compactor/l1.py:51-60` 부근 `_schema_version(channel: str) -> str` 함수 = channel → schema_version mapping SSOT. 현재 `("transaction" → "tick.v1", "orderbooksnapshot" → "orderbook_snapshot.v1")` 2 entry, else `NotImplementedError`. `compact_segment` (sealed segment lifecycle 처리 entrypoint) 가 본 함수 호출 → unsupported channel 시 silent catch (outer try/except, Prometheus alert 0).
2. **WAL sealed lifecycle**: collector ingester (`bithumb-ingester` / `upbit-ingester` service) 가 channel 별 NDJSON segment 를 `/var/lib/mctrader/data/wal/{exchange}/{channel}/` 에 append → segment full 시 sealed (rotate) → compactor (`mctrader-compactor` service) 가 periodic poll 로 sealed file 읽고 L1 parquet 변환 → sealed file delete. **본 Story scope = 변환 단계 (sealed → L1 parquet) 의 channel allowlist 확장**.
3. **WAL payload schema mismatch** (3 channel):
   - `transaction` = flat trade row (1 frame = 1 trade event = 1 row)
   - `orderbooksnapshot` = per-level full snapshot (1 frame = 60 levels = 60 rows fixed, §D14)
   - `orderbookdepth` = **delta `changes` event** (1 frame = N levels 가변, qty=0 = level delete) — 본 Story 의 신규 schema 영역
4. **bithumb 만 emit, upbit emit 0** (CodebaseMapperAgent + OperationalRiskArch deputy 통합 박제): upbit collector WAL sample 실측 (`ls /var/lib/mctrader/data/wal/upbit/`) = `orderbooksnapshot` / `transaction` only, **`orderbookdepth` 0**. 본 사실은 본 Story 의 fix 가 upbit L1 lost 의 진짜 root cause 가 **아님** 의미 — Story §6 D9 + R4 박제 surface.
