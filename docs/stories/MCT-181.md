## §6 Risk 상세

### R-MCT-181-1 image-publish.yml cross-repo checkout 권한 (HIGH)

- **위협**: MCTRADER_CROSS_REPO_TOKEN scope 부족 시 sibling repo (mctrader-data / mctrader-engine /
  mctrader-signal-collector) checkout + Docker build 실패
- **mitigation**: MCT-176 LAND `MCTRADER_CROSS_REPO_TOKEN` secret (hub repo owner) 재사용.
  strategy.matrix 3 repo 각 checkout `token: secrets.MCTRADER_CROSS_REPO_TOKEN`. Phase 2 PR1
  workflow_dispatch dry-run 검증 의무 (AC-2).

### R-MCT-181-2 backtest artifact NAS sync 재실행 idempotency (MEDIUM)

- **위협**: backtest 재실행 시 동일 run_id 로 NAS sync → partial overwrite
- **mitigation**: completion marker `.done` 존재 시 sync skip (idempotent). 새 backtest = 새 run_id
  (timestamp-based UUID). `.done` marker 없는 run_id 만 retry.

### R-MCT-181-3 ${IMAGE_TAG} prod pin 미적용 시 dev=latest 폴백 (LOW)

- **위협**: .env.prod 에 IMAGE_TAG 미설정 시 `:-latest` fallback → prod 환경 latest pull (pin 의도 위반)
- **mitigation**: `.env.prod.example` 에 `IMAGE_TAG=<release_tag_or_git_sha>` 주석 의무 박제.
  operator는 prod deploy 전 IMAGE_TAG 명시 의무. ${IMAGE_TAG:-latest} 는 dev 안전 (dev=latest 허용).
