### 변경 대상 ADR

**ADR-009 (OHLCV schema) — Major amendment**:
- Candle 정의를 stored entity 에서 **derived view** 로 격하
- tick.v1.1 minor extension: `ingest_seq` (uint64) + `payload_hash` + `validation_status` 추가
- `provenance` column 추가 (`legacy_candle` / `transaction_derived`)
- Information bar contract 정의 — label format: `time_5m` / `vol_1000000krw` / `tick_1000` / `dollar_5000000krw`
- immutable contract metadata: genesis_ts / threshold / precision / version / contract_id (hash)

**ADR-017 (zero-loss ingestion + WAL + tiered compaction) — amendment**:
- transaction-tier WAL 정책: at-least-once + batch fsync 10-100ms 또는 1000 msg
- WAL memory buffer 50,000 msg (5000 msg/sec × 10sec burst)
- SLA: power-loss window ≤ 100ms or ≤ 1000 msg (먼저 도달)
- Compactor MCT-132 framework 확장 — transaction-tier policy
- fallback tuple dedup reference (ADR-009 D10.7)

**신규 ADR — Aggregation Core Lib Contract**:
- Hot(asyncio) / Cold(DuckDB) 양 path shared pure-Python core
- time + volume/tick/dollar bar SSOT
- immutable contract metadata 발급
- Hot/Cold consistency SLO: drift < 0.01%

**신규 ADR — Legacy Candle Provenance & Retirement Policy**:
- cutoff timestamp (strategy retro 가능 시점 = month boundary)
- 2-4주 dual-write 검증
- reconciliation exit criteria
- candle collector retirement 절차

### WHY (Epic 1차 동기)

비표준 timeframe 유연성 — 시간 bar (임의 timeframe) 외 volume/tick/dollar bar 등 **information-driven bar** (Lopez de Prado AFML) 생성 자유도 확보. candle = derived view 격하로 단일 SSOT 단순화.

### §7.4 Schema 변경 (tick.v1 → tick.v1.1 minor)

- **Minor extension** (ADR-009 §D7 compatible 정의 정합) — 3 column 추가:
  - `ingest_seq` (uint64) — collector 측 monotonic 발급
  - `payload_hash` (string) — raw WS frame SHA256 16-hex prefix
  - `validation_status` (string) — `OK` / `GAP` / `MALFORMED` / `RECONNECT_BOUNDARY`
- **Backward compat** — 기존 tick.v1 Parquet (legacy) → v1.1 reader 가 신규 column NULL 또는 `OK` default 처리. legacy row 도 immutable 유지.
- **Partition layout 변경 없음** — `market/ticks/schema_version=tick.v1/...` 유지 (minor 는 partition label 변경 트리거 아님, ADR-009 §D7).
- **Story-4 implementation seal** — 신규 row 부터 3 column non-null write 의무.

### §7.8 Security (SecurityArchitect perspective 합성)

- **Bithumb WS frame validation** — `payload_hash` (raw frame SHA256) 박제로 frame integrity 검증 가능. malicious frame injection 시 hash mismatch → quarantine + `validation_status="MALFORMED"`.
- **GAP / MALFORMED 박제 의무** — Bithumb upstream 의 invariant 위반 (sequence hole, schema mismatch) 시 row-level visibility 의무화. silent skip 거부 → audit trail 확보.
- **raw_json 보존 정책** — `raw_json` nullable (debug optional) 유지. 단 active-active dedup mismatch 시 raw_json 보존 의무 (forensic). production 시 disk pressure 발생 시 raw_json prune 가능 (Story-6 implementation 정책).
- **Trust boundary** — Bithumb WS endpoint = external (untrusted), collector → WAL = trust boundary crossing. WAL fsync + payload_hash 가 boundary integrity 의무.
