## §1 동기

**EPIC-mctrader-docker-stack Story-3 (sequential_phase 3)** — compose stack 에 두 번째 어플리케이션 service
추가. MCT-176 LAND (hub#330 + data#64 + hub#331 + Phase 2 PR2) 로 `mctrader-data collector` 컨테이너가
완성됨. 본 Story 는 그 위에 `mctrader-engine paper-engine` daemon 을 compose service 로 진입.

4 결정 구현:

1. **D2 paper daemon (restart unless-stopped)** — compose.yml 에 `paper-engine` service 신규
   (image: ghcr.io/mclayer/mctrader-engine, command: `paper --daemon`, restart: unless-stopped,
   healthcheck :8080, stop_grace_period: 60s). backtest-runner = MCT-178 (별도 profile, D2 oneshot 부분).

2. **D4 SIGTERM graceful + startup InvariantHarness scan** — mctrader-engine CLI 측
   `signal.SIGTERM` / `signal.SIGINT` handler 신규. 60s grace period 내 open position commit
   + WAL flush + exit 0. compose stop_grace_period 정합. startup 시 InvariantHarness 8종 scan
   (MCT-171 SSOT, 위반 시 warn + continue).

3. **D10 universe override** — env default `UNIVERSE_TOP_N=50` (.env.dev / .env.prod) 박제.
   compose command `--universe-id <id>` override (backtest case 포함). paper daemon 측
   50-sym universe default.

4. **D15 Redis key prefix** — Redis key namespace 분리:
   - `signal:*` (signal-collector 5종 — fear_greed / ecos / kimchi / announcement / coinglass)
   - `market:*` (mctrader-data tick + orderbook cache)
   - `engine:*` (paper-engine position + strategy state)
   migration: signal-collector 5종 기존 unprefixed key → `signal:*` rename (1주일 dual write 후 cutover).

## §6 Risk 상세

### R-MCT-177-1 Redis prefix migration silent fail (MEDIUM)

- **위협**: signal-collector 5종 unprefixed key → `signal:*` rename 중 consumer (paper-engine) 가
  old key 로 read 시도 → 데이터 없음 (KeyError / null) 상황 무통보 발생
- **mitigation**:
  - 1주일 dual write (legacy unprefixed + `signal:*` 동시 write)
  - paper-engine 측 `signal:*` prefix 우선 read, fallback 없음 (dual write 기간 중 항상 `signal:*` 존재)
  - dual write 종료 시점: MCT-177 LAND + 7d 이후 별 PR 에서 legacy key cleanup
  - Prometheus `redis_key_migration_dual_write_active` flag Gauge (1=dual write 활성)
- **잔여 risk**: dual write 기간 내 compose 재시작 시 signal-collector 가 구버전 image 로 기동 =
  unprefixed key 만 write → paper-engine `signal:*` read miss. image tag pin (MCT-181) 전까지 주의.

### R-MCT-177-2 paper daemon position state loss on crash (MEDIUM)

- **위협**: paper-engine SIGTERM 수신 후 60s grace 내 open position commit 미완료 → restart 시
  position state 불일치 (`engine:position:*` Redis key stale)
- **mitigation**:
  - 60s grace = SIGTERM 수신 → open position 즉시 snapshot write → WAL flush → exit 0 순서 보장
  - restart unless-stopped 로 자동 재시작 후 Redis `engine:position:*` 읽어 state 복구
  - InvariantHarness startup scan: 8종 ambiguity invariant 검증 후 trading loop 진입
  - position snapshot = Redis atomic SET (SETNX 아님, restart 시 최신 snapshot 덮어쓰기 허용)
- **잔여 risk**: Redis container 자체 crash 시 position state 영구 손실. Redis AOF persistence = MCT-179
  에서 observability 설정 시 enable 여부 결정 예정.

### R-MCT-177-3 universe-id override misconfig (LOW)

- **위협**: compose command `--universe-id` override 값이 engine 내부 registry 에 없는 ID →
  paper daemon startup fail (KeyError) 또는 empty universe 로 trading loop 진입
- **mitigation**:
  - CLI 측 universe-id validate: registry 미존재 시 즉시 exit 1 + error 메시지
  - UNIVERSE_TOP_N=50 env default = 50-sym universe (MCT-103 LAND, 항상 유효)
  - compose command override 는 operator explicit action — runbook 박제 (AC-3 verify)
