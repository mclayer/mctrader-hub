# MCT-103: 50-symbol universe expansion (backtest history accumulation)

- **Status**: in_progress
- **Trigger**: 단타 + 백테스트 history 누적 의무 — forward-only invariant (ADR-009 §D12) 하에 collect 시작 시점 = data history 시작 시점
- **Parent Epic**: MCT-98 (Docker-first Migration, CLOSED) follow-up — operational expansion

## 1. 사용자 요구사항 (verbatim, 2026-05-09 session)

> "거래 가능성이 있는 코인은 이전부터 데이터를 모아야한다. backtesting을 하기 위함이다. 50코인정도는 모아도 되지 않을까? 이 코인을 선정하기 위한 점수표를 생성해서 점수를 매겨 결정해보자"
> "이것까지 시작하자."

배경: F7 smoke (MCT-98 Phase 6 post-close) 후 사용자가 단타 + 백테스트 누적용 universe 확장 결정. 현재 collector 의 `--top-n 10` (24h volume top 10) 한계 해소. 단순 top-N 부족 — spread / depth / volatility / stature / tick freq 가중 점수표 필요.

## 2. 도메인 해석

### 2.1 Forward-only invariant 의 의미

ADR-009 §D12: tick / orderbook = forward-only, backfill 불가. 즉:

- **collect 시작 시점이 backtest history 시작 시점**
- **하루 늦게 시작 = 그 하루는 영구 손실**
- universe rebalance 시 drop 된 종목 데이터는 보존 (다시 부활 시 history gap 존재)

→ 결정에 시간 들이지 말고 결정한 universe 즉시 collect 시작 의무.

### 2.2 점수표 6 dimension (총 100점)

각 항목 stepped 함수:

| 항목 | 배점 | 단타용 의미 | 백테스트용 의미 |
|---|---:|---|---|
| 24h KRW 거래대금 | 25 | slippage | data 부피 |
| Order book 10-level depth | 15 | execution size | tick replay 정합 |
| Spread bps | 20 | scalping 경제성 | mkt-making backtest |
| Volatility (H/L%) sweet spot 곡선 | 20 | momentum 알파 | RSI/ATR 시그널 빈도 |
| Stature (majors bonus) | 10 | 유동성 안정성 | 장기 history 신뢰성 |
| Tick frequency proxy (units/24h) | 10 | 진입/청산 기회 | tick-resolution 전략 |

Stage functions:

```
Volume:    >=500억=25 / 200-500=20 / 100-200=15 / 50-100=10 / 10-50=5 / <10=0
Depth(M):  >=1000=15 / 500-1000=12 / 200-500=9 / 100-200=6 / 50-100=3 / <50=0
Spread:    <=5=20 / 5-10=16 / 10-20=12 / 20-40=8 / 40-80=4 / >80=0
HL%:       3-8=20 (sweet) / 8-15=18 / 15-25=14 / 1-3=10 / 25-50=8 / <1=4 / >50=2
Stature:   BTC/ETH/SOL/XRP/...=10 / Korean popular=6 / 그 외 vol>=500억=4 / 0
Units:     >=1B=10 / 100M-1B=8 / 10M-100M=6 / 1M-10M=4 / 100k-1M=2 / <100k=0
```
