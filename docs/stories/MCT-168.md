# MCT-168 — L1 NAS DualWriter wiring (EPIC-tier-promotion-single-source Story-2)

## §1 동기

**EPIC-tier-promotion-single-source Story-2** — ADR-029 D1+D2 owner Story. L1 ParquetWriter atomic 직후 DualWriter.put_streaming() 호출 + retry_queue 재사용으로 L1 NAS dual-write hot path 확립.

### 1.1 직접 동기

MCT-167 (ADR-029 publish) 가 결정한 D1=B (L1 NAS PUT timing = L1 ParquetWriter atomic 직후, compactor 측) + D2=B (DualWriter retry_queue + local_only 재사용) 을 구현. ADR-027 §D5 "L1 NAS upload 금지" invariant 폐기 후 첫 번째 구현 Story.

NFR: L1 PUT p99 < 1500 ms (NAS LAN baseline + MCT-148 T2 ± 15%).

### 1.2 D1+D2 결정 (ADR-029 박제분)

- **D1=B**: L1 NAS PUT timing = L1 ParquetWriter atomic 직후 (compactor 측). 파일 단위 정합성 + hot path 지연 균형.
- **D2=B**: NAS PUT 동기/비동기 = 기존 DualWriter retry_queue + local_only 재사용. hot path block 최소 + MCT-150/151 primitive 재사용 (logical sync).
- **NFR**: L1 PUT p99 < 1500 ms (NAS LAN baseline + MCT-148 T2 ± 15%)

## §2 Phase 0 Context (brainstorm 답습)

별 세션 (2026-05-14) PMOAgent 2nd pass 박제분.

### 2.1 AC (6~8)

- **AC-6 (L1 NAS PUT trigger)**: compactor L1 segment atomic rename 직후 DualWriter.put_l1() 호출 + Prometheus `mctrader_dual_write_result_total{tier="L1",status}` emit
- **AC-7 (retry_queue + local_only)**: NAS PUT fail (max retry 소진) → retry_queue append + status=local_only 반환, compactor 정상 종료 (L1 local 보존)
- **AC-8 (L1 PUT p99 < 1500ms NFR)**: 100회 측정 p99 < 1500 ms, Prometheus histogram `mctrader_dual_write_l1_latency_seconds` 검증

### 2.2 INV (4~6)

- **INV-4**: L1 ParquetWriter atomic rename 성공 = L1 local SSOT (NAS PUT 실패해도 segment 보존)
- **INV-5**: DualWriter.put_l1() status enum 3종 (committed / local_only / hard_floor_blocked) 정확 1개 반환
- **INV-6**: retry_queue replay idempotent (NAS versioning + Object Lock 의존, MCT-161 정합)

### 2.3 Risk (3)

- **R-1 (HIGH)**: L1 PUT p99 NFR 미달 → 완화 = MCT-163 put_streaming 적용 + segment size budget + async PUT fallback escalate
- **R-2 (MID)**: retry_queue overflow → 완화 = max size cap + Prometheus alert + manual drain runbook
- **R-3 (MID)**: L1 mode + L2/L3 mode object key 충돌 → 완화 = tier prefix enforce (l1/ l2/ l3/) + path schema test
