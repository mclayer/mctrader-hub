## 1. 배경

ADR-017 (Zero-loss ingestion + WAL → L3 tiered compaction) 정착 이후 L3 Parquet 객체가 MinIO `mctrader-market` 버킷에 누적되기 시작. 그러나 mctrader 6 repo 어디에도 L3 Parquet 에 ad-hoc SQL 을 실행할 수 있는 도구가 없어:
- 데이터 검증 (백필 누락 / OHLCV 시계열 gap) 시 매번 임시 Python 스크립트 작성
- 운영 dashboard / Grafana 가 직접 Parquet 읽을 수 없음 → 별도 ETL 필요
- 백테스트 외부에서 시장 데이터 query 수단 부재

DuckDB 의 `httpfs` extension 으로 S3 호환 객체(MinIO)를 직접 SQL 로 query 가능 — read-only 분석 레이어 도입.

## 3. 핵심 설계 결정

- **Read-only 정책**: `DuckDBReader` 는 `read_only=True` 옵션으로 connection 생성 — write 차단으로 L3 Parquet 손상 방지
- **MinIO config injection**: `MinioConfig(endpoint, access_key, secret_key, region, use_ssl)` dataclass 가 `SET s3_endpoint='...'` 등 DuckDB SET 명령을 f-string 으로 생성 — 환경변수 기반 인프라 설정값으로 SQL injection 위험 허용 수준 (§3.3 참조)
- **CLI 출력 포맷**: `result.df()` 가 numpy 의존 → 최소 의존성 유지를 위해 `cursor.fetchall() + cursor.description` 기반 수동 텍스트 테이블 출력 (§3.2 참조)
- **mctrader-engine 의존성만 추가**: 본 Story 에서 engine 측 사용 코드는 미작성 — 향후 백테스트 사후 분석 / 운영 분석 컴포넌트 추가 시 import 가능하도록 의존성만 박제
