## §1 동기 (배경/문제)

사용자 원문: *"data 영역은 mctrader와 아예 분리하고 싶다. 표준 mctrader에는 호출용 interface,
data에는 그를 처리하는 REST API가 있는게 좋겠다. 프로젝트 구조는 mctrader-hub에서도 구현할 수
있고 mctrader-data 단독으로도 필요사항을 반영할 수 있어야겠다."*

**EPIC-data-domain-decoupling Story-1 (sequential_phase 1, Epic 진입 Story)**. Epic 종착점 =
mctrader-engine 을 (1) data-free (`mctrader_data` python 의존 0) + (2) exchange-agnostic
pure consumer (`mctrader_market_bithumb/upbit/*` 어댑터 직접 의존 0) 로 전환. 사용자 첫 요구
"신규 거래소 data 단독 반영" 의 다중거래소 일반화 + 4-Layer 재설계 (Phase 1 dialog 확정).

### 문제 (Phase 0 deep-verify 실증)

현재 `mctrader-engine` 이 `mctrader_data` 내부 모듈을 python 으로 직접 import (engine→data
14 import 줄). 그중 본 Story 대상 = **PURE-contract 부분** — 거래소·storage 결합이 없는
순수 도메인 계약 코드인데도 `mctrader_data` namespace 에 거주해 engine→data 결합의 일부를
형성한다:

- `mctrader_data.aggregation` (4 bar 알고리즘 SSOT — TimeBar/Volume/Tick/DollarBarAggregator
  + ContractMetadata + scaled-int helper) — **PURE** (의존 = `mctrader_market.{protocols.information_bar,
  schemas.tick, types}` + stdlib 뿐, `mctrader_data.*` 0)
- `TickRecord`/`OrderbookEventRecord` 순수 dataclass — pyarrow writer 모듈(`tick_storage.py`/
  `orderbook_storage.py`)에 **동거**하나 dataclass 본문은 stdlib only (pyarrow 비결합)
- `paper_lineage` (`PaperLineage` / `canonical_jsonl_hash`) — **PURE** (stdlib + pydantic +
  `mctrader_market.types` 뿐)
- `CandleModel` — **이미** `mctrader_market.candle` 에 존재. engine 의 `from
  mctrader_data.cold.duckdb_resample import CandleModel` (실측 4곳) 은 `duckdb_resample.py` 가
  `mctrader_market.candle` 를 단순 re-export 하는 경로 → engine 이 data 를 경유할 이유 0

### 추출된 동기 (spec §1 확정, 4종)

| why | 내용 |
|---|---|
| 코드 결합도 감소 | engine 이 `mctrader_data` 내부 모듈을 직접 import 하는 구조를 끊고 안정된 계약 표면으로만 통신 |
| 독립 배포·버전 관리 | data 를 engine 릴리스와 무관하게 배포/롤백/스케일 |
| 외부·다중 consumer | engine 외 시스템도 data 를 표준 API 로 호출 |
| 다중거래소 확장 | 신규 거래소 추가가 **data 단독 변경**으로 끝나야 함 (사용자 핵심 지시) — Layer0 contract 안정화가 그 선행 조건 |

본 Story(MCT-182) = strangler-fig **1단계**. Layer 0 (mctrader-market = FOUNDATION,
의존 0, DAG 최하위) 을 확립한다. PURE-contract 3종 + CandleModel 경로 정리를 market 으로
이전·재지정하고 data/engine 호출부는 무중단 deprecation shim 으로 전환. 후속 Story
(MCT-183~188) 의 io relocate / REST boundary / engine cutover 가 안정된 Layer0 위에서
진행되도록 토대를 놓는다. + **ADR-031 publish (Proposed)**.
