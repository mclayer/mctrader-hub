### 문제 (Phase 0 deep-verify 실증)

MCT-183 LAND 결과 — engine cold-tier read 자산 (io/ 6 module: `tier_reader` /
`reader_cache` / `endpoint_router` / `dr_mode` / `cold_reader` / `l1_reader`) 이
mctrader-data 측 `src/mctrader_data/io/` 로 물리 이전됨 (Layer2 DATA-STORAGE 영역 단독
소유, §0 V5). 그러나 현 상태 — engine 의 실제 cold-read 는 여전히
`mctrader_data.storage.scan_candles` / `orderbook_replay` /
`path.resolve_data_root` 를 **python 직접 import** (= D2 cold-read-behind-REST 절반,
MCT-185 cutover 대상). 즉 **engine→data python 결합이 잔존**한다.

문제의 본질: **engine 이 data 를 표준 계약 표면(REST)이 아니라 python import 로 호출**한다.
사용자 핵심 요구 "표준 mctrader에는 호출용 interface, data에는 그를 처리하는 REST API" 는
data 측 **호출 가능한 REST API 표면 부재**가 미충족 상태다 (§0 V2/V3: data web framework
0, HTTP = `health_server.py` stdlib liveness probe only — REST API 부재). engine pure
consumer 전환(Layer2')의 선행 조건 = **data 가 io/ reader 자산을 REST 뒤로 노출**해야
한다 (engine 이 data 내부 모듈을 import 하는 대신 안정 계약 표면으로만 통신).

본 Story(MCT-184) = strangler-fig **3단계 일부** (REST 신설 — cutover 전제). MCT-183 이
물리 이전한 io/ 6 module (dead-in-data) 을 FastAPI `/v1` REST API 뒤로 노출하고, 역방향
write POST (paper-candles / backtest-artifact) 를 신설한다. **engine cold-read 실경로
cutover (engine `mctrader_data.storage.*` 직독 → data REST 경유) + realtime 정규화 stream
(Redis Stream) + engine `data_client/` thin client 는 MCT-185 owner — 본 Story 는 REST
boundary 신설 (historical + reverse-write) + OpenAPI SSOT + hub contract snapshot
governance + ADR-030 amendment box only**.

### presigned-NAS-handoff 기각 재명시 (D2/ADR-029 정합)

spec §5 "Codex 리뷰 합성 — 기각 1": Codex 가 D3/D5 에서 권장한 **presigned NAS handoff**
(data 가 NAS presigned URL 발급 → engine 이 NAS 직접 fetch) 는 **기각**한다. 사유:
"code import만 0" 으로 data-free 재정의 → engine 이 NAS 객체 레이아웃·parquet·tier·DR
지식 보유 잔존 = 사용자 의도("데이터 전부 제거" + 외부 consumer) + β 위반. 검증상 io/
미배선(MCT-183 dead-in-data) + 실시간 hot-path data 비의존(거래소 WS) + ADR-030
single-host loopback → presigned 가 풀려던 성능 문제 비실재. **Arrow IPC over REST** 채택
(engine 은 NAS object layout / parquet tier / ETag / endpoint resolution **비인지** —
data 가 io/ reader 로 NAS 접근, REST 응답은 Arrow IPC stream only). 이는 D2
(cold-read-behind-REST) + ADR-029 (NAS SoT = data 단독 소유) 정합 — engine NAS 직독 폐기의
선행 토대.
