## 2. Fix 브랜치 현황

### `fix/engine-backtest-equity`
| 항목 | 내용 |
|---|---|
| `executor/backtest.py` | `_step()` 루프 순서 교정: `_record_equity(current_candle)` **먼저** → `_submit_and_fill()` **나중** (time-travel 제거) |
| `executor/backtest.py` | `_snapshot_portfolio()`: `gross_exposure = position_quantity × candle.close` + `mark_price = candle.close` RiskGate에 주입 |

### `fix/engine-paper-equity`
| 항목 | 내용 |
|---|---|
| `executor/paper.py` | `_build_equity_rows()`: FILLED OrderEvent를 ts_utc 순으로 적용 (BUY: cash-=cost+fee, position+=qty; SELL: cash+=proceeds-fee, position-=qty) |
| `executor/paper.py` | FILLED 이벤트 emit: `side`, `notional`, `fee` 필드 추가 (ADR-004 parity) |

### `fix/engine-wfo-promote`
| 항목 | 내용 |
|---|---|
| `wfo/cli.py` | `promote` 명령에서 `--gate-d6-passed`, `--l4-clean`, `--dsr-significant` 플래그 **제거** |
| `wfo/cli.py` | `gate_d6_result.json`, `audit.log`, `corrected_metrics.json` 아티팩트에서 promote 조건 유도 |
| `wfo/evaluator/coordinator.py` | `run_evaluate()` 후 `gate_d6_result.json`, `corrected_metrics.json` 기록 |

### `fix/engine-db-risk-account`
| 항목 | 내용 |
|---|---|
| `db/run_store.py` | `_extract_side()`: default "buy" 제거, fill event에서 side 조회, 없으면 `ValueError` |
| `db/run_store.py` | FILLED 이벤트 처리: `orders.quantity`, `orders.price` fill 값으로 업데이트 |
| `risk/rate_limiter.py` | `try_consume_order_create()`, `try_consume_cancel()` 원자 메서드 추가 (threading.Lock) |
| `account/virtual.py` | `fill()`: BUY 시 `InsufficientFundsError`, SELL 시 `InsufficientInventoryError` |
