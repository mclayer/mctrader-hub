## §2 도메인 해석 (DomainAgent 통합)

### 내부 제약 (mctrader 6-repo 도메인 해석)

**관찰 1 — ADR-017 후속 link (cold tier extension of zero-loss invariant)**: ADR-017 (Zero-loss ingestion via WAL + tiered compaction) 은 hot path (Ingester WAL + Compactor L1/L2/L3 tiered) 의 zero-loss invariant 박제. 본 ADR-027 은 그 **cold tier (L2/L3 compacted Parquet) 의 external storage layer**. ADR-017 § Decision (D3 — L1/L2/L3 tier layout) 의 cold tier 정의를 외부 NAS object storage 로 확장하되, hot path (collector WAL + L1) 는 local volume 유지 의무 (zero-loss invariant 보존). 본 ADR-027 = **ADR-017 의 successor (확장)**, supersede 아님 — ADR-017 의 hot durability 정책은 유지, cold tier storage backend 만 추가 정책.

**관찰 2 — ADR-016 complement (immutable audit log + cold tier archive policy 정합)**: ADR-016 (Audit log append-only with hash chain) 은 mctrader-web `admin_audit.sqlite` 의 forensic-grade append-only + WAL 정책. ADR-016 §A4 "NFS/SMB/network filesystem 위 named volume 금지" 제약은 POSIX fsync 의존성에 기반 — 본 ADR-027 의 NAS MinIO 는 **application-layer S3 API** (boto3 over HTTP, POSIX semantic 의존 0) → ADR-016 제약 정합 회피. ADR-016 의 admin_audit.sqlite cold archive (별도 Epic) 는 본 ADR scope 외 (D11 명시 — admin_audit.sqlite immutable archive 운영 모델은 sqlite WAL ↔ parquet cold archive 상이) — ADR-027 은 **complement** (보완), supersede 아님.

**관찰 3 — ADR-009 SSOT extension (`MCTRADER_DATA_ROOT` + `NAS_MINIO_*` namespace 분리)**: ADR-009 §D12.1 (Docker-first persistence) 는 `MCTRADER_DATA_ROOT` 환경변수 + Docker named volume `mctrader_data` 를 storage path SSOT 로 박제. 본 ADR-027 의 NAS MinIO 는 이 SSOT 의 **cold tier extension** — hot path env (`MINIO_ENDPOINT` = local minio container) 와 NAS env (`NAS_MINIO_*` 별도 namespace) 분리 의무. MCT-148 의 test fixture 가 이 namespace 분리를 이미 검증 (test 가 `MINIO_ENDPOINT` 침범 0). MCT-155 cutover 시 endpoint swap = `MINIO_ENDPOINT` 가 NAS endpoint 로 전환 (`NAS_MINIO_*` 는 deprecate) — 본 ADR §Decision D4 cutover scope.

**관찰 4 — ADR-008 secret management 정합 (90일 rotation runbook)**: ADR-008 §D7 (Rotation = event-driven + 정기, 반기 1회 / 분기 점검) 의 cadence 와 본 ADR-027 의 90일 rotation cadence 는 정합 (MinIO root credential = secret store item, hub `.env` gitignored). MCT-147 의 `nas-minio-secret-rotation.md` runbook 이 ADR-008 의 8-step rotation procedure 의 cold-tier-specific 실행 박제. Emergency rotation (compromise) 시 ADR-008 §D8 (Compromise emergency response) 의 6-step + cold tier endpoint cutover 정합 의무.

**관찰 5 — ADR-027 reservation marker 박제 → 본 Story 본문 land**: 사전 박제 marker (`docs/adr/.reservation-ADR-027.md`) 가 D1~D11 + relationships (successor_of=[ADR-017], complements=[ADR-016], references=[ADR-008, ADR-009]) 박제. 본 Story 가 ADR 본문 author 시 marker §removal 정합 — marker file `Removal` field ("MCT-149 land 시 본 marker 삭제 + 본문 land") 명시.

### 도메인 용어 정합

- **cold tier**: L2 (1h aggregation compacted Parquet, ADR-017 §3 D3) + L3 (1day aggregation, long-term archive). hot tier (collector WAL + L1 5min Parquet) 와 storage backend 분리.
- **NAS MinIO direct deploy**: WebDAV mount 우회 (NFS/SMB filesystem mount 위에 minio = `format.json` atomic op 실패, minio/minio#14060 unsupported). NAS Synology Container Manager (DSM 7.2+) 위에 MinIO 컨테이너 직접 deploy + NAS local btrfs/ext4 volume 을 MinIO data dir 로 mount = application-layer S3 API 만 사용, POSIX semantic 의존 0.
- **bucket layout (단일 mctrader-market + Hive prefix)**: 기존 `minio_uploader.py` 의 `BUCKET_NAME='mctrader-market'` 하드코딩과 일치 → MCT-155 cutover 시 코드 변경 없이 endpoint swap. Hive prefix = `schema_version/exchange/node/tier/date` (ADR-009 D2 + ADR-017 D3 정합).
- **dual-write window + atomic switch**: MCT-150~155 의 cutover 단계 — local volume write + NAS PUT 동시 수행 → 3종 invariant PASS → reader endpoint NAS 로 전환 → 7일 grace 후 local GC. 본 ADR §Decision D4 박제 scope.
- **3종 invariant (sha256 + object count + parquet row count)**: ADR-009 16-col schema 검증 + forward-only invariant 정합. MCT-148 T3 (sha256 IDENTICAL 3/3) + T5 (atomic_invariant=true) 가 invariant 의 measurable 측면 사전 입증.
- **MCT-148 evidence pack**: gitignored test artifact (`c:/workspace/mclayer/mctrader-data/.tmp/evidence-pack-MCT-148.md`). 5 PoC raw json + DevPL §종합 요약 + T4 manual gate 추가 박제. 본 ADR §Decision D8 의 evidence 직접 인용 source (transcribe — gitignored 이므로 ADR 본문에 raw json inline).

### 상충·정합 분석 (3 ADR cross-link 검증)

- **ADR-017 ↔ ADR-027** = successor (확장). ADR-017 §3 D3 의 cold tier (L2/L3) storage backend 를 local → NAS object storage 로 확장. hot path (ingester WAL + L1) = ADR-017 정책 유지. 충돌 0.
- **ADR-016 ↔ ADR-027** = complement (보완). ADR-016 §A4 의 "NFS/SMB 위 named volume 금지" 는 POSIX fsync 의존 file system layer 제약 — 본 ADR-027 의 application-layer S3 API 는 정합 회피. admin_audit.sqlite 와 cold tier parquet 의 storage backend = 별 Epic (D11). 충돌 0.
