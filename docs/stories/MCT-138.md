### 변경 내용

**Transaction WS Subscriber 강화** (`mctrader_market_bithumb/subscribers/transaction_ws.py`):
- 기존 ws_subscribe.py / ws_mapping.py / ws_events.py 의 transaction event 처리 확장
- 추가 field 생성:
  - `ingest_seq` (uint64) — monotonic, collector start 부터 누적
  - `payload_hash` (string) — raw WS frame SHA256 16-hex
  - `validation_status` (string) — 정상 처리 시 `OK`
- WS reconnect 검출 시 `validation_status=RECONNECT_BOUNDARY` 박제
- gap detection: sequence hole 또는 expected timestamp 큰 jump 감지 시 `validation_status=GAP` + alert emit
- malformed payload (schema mismatch / 음수 price/qty / unknown side) → `validation_status=MALFORMED` + quarantine

### Gap detection 정책 (CRITICAL)

ADR-009 D12.2: Bithumb public API 는 ticks historical replay 미제공. gap 검출 시:
- alert emit (사용자 통지)
- REST `/public/transaction_history` snapshot 시도 (best-effort, recent 20-100건만 가능)
- 실패 시 `validation_status=GAP` 영구 박제
