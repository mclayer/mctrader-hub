# MCT-124: mctrader-signal-collector — 외부 시그널 수집 서비스

- **Status**: 완료
- **Trigger**: mctrader 엔진이 OHLCV/tick 데이터만 보고 결정 — 외부 컨텍스트(공포탐욕지수, 한국 리테일 수급, 글로벌 파생 압력, 거래소 공지) 부재
- **Repos**:
  - `mctrader-signal-collector` (신규) — 외부 시그널 4종 수집, Redis Stream publish
  - `mctrader-hub` — compose.yml signal-collector 5 서비스 추가, prometheus.yml 5 scrape jobs 추가

## 1. 배경

mctrader는 현재 Bithumb + Upbit OHLCV / WebSocket tick만 수집. 거래 판단에 필요한 외부 컨텍스트가 없으면 엔진이 가격 패턴만 보고 결정.

본 Story는 Tier 1 외부 시그널 4종을 `mctrader-signal-collector` 신규 독립 서비스로 수집해 Redis Stream(`signals:{kind}`)에 publish. Engine은 Redis Stream을 subscribe해 포지션·리스크 결정에 반영.

**채택 접근법 — 별도 `mctrader-signal-collector` 서비스 (접근법 B)**
- 소스별 독립 컨테이너 → 장애 격리
- Engine과 완전 decoupled (Redis Stream 인터페이스만 공유)
- stale 데이터 시 `stale=true` neutral 메시지 publish → engine fallback

## 4. 핵심 설계 결정

- **독립 리포**: 장애 격리 + Engine 완전 decoupled (Redis Stream 인터페이스만 공유)
- **stale fallback 의무화**: upstream 실패 시 `stale=true` neutral publish — engine이 stale=true 시 시그널 무시
- **instance-level CollectorRegistry**: 테스트 다중 인스턴스 시 전역 registry 충돌 방지
- **_CLASSIFY_MAP specific-first 원칙**: "상장폐지" → "상장" 순서 — substring 충돌 방지
- **kimchi premium 공개 API 전환**: CryptoQuant 무료 플랜 403 → Upbit BTC/KRW + Binance BTC/USDT + Frankfurter USD/KRW 조합으로 API 키 없이 계산
- **Upbit API**: `api-manager.upbit.com/api/v1/announcements` 내부 API (httpx 호환)
- **Bithumb API**: `feed.bithumb.com` Cloudflare-protected Next.js SPA → cloudscraper + buildId 추출 → `/_next/data/{buildId}/notice.json`
- **ECOS 주말 공백 대응**: 7일 범위 쿼리 + 항목코드 0000001(USD) 명시 → 가장 최신 발표일자 선택

**런타임 발견 이슈 & 해결**:
1. **Upbit/Bithumb React SPA 문제**: 브라우저 DevTools 대신 GitHub code search로 실제 내부 API 발견
   - Upbit: `api-manager.upbit.com/api/v1/announcements` (Korean-locale Accept-Language 헤더 필요)
   - Bithumb: `feed.bithumb.com` Cloudflare 차단 → cloudscraper + buildId 추출 → Next.js data API
2. **CryptoQuant 403**: 무료 플랜은 `korea-premium-index` 엔드포인트 미지원 → Upbit + Binance + Frankfurter 공개 API 조합으로 대체 (API 키 불필요)
3. **ECOS 주말 공백**: `today` 단일 날짜 쿼리는 주말/공휴일에 empty → 7일 범위 + 항목코드 0000001 추가
4. **Frankfurter API 이전**: `api.frankfurter.app` → `api.frankfurter.dev/v1` (301 redirect 발생, 직접 수정)
5. **cloudscraper 버전**: `>=2.3` 없음 (max 1.2.71) → `>=1.2` 제약으로 수정
