# MCT-170 — Engine reader L1 확장 + DR mode + reader cache byte budget (EPIC-tier-promotion-single-source Story-4)

## §1 동기

**EPIC-tier-promotion-single-source Story-4** — ADR-029 D7=A + D8=B owner Story. engine reader 측에 L1 tier 인지 + D7 cache NFR (95% hit + p99 <100ms) + D8 local fallback (forward-only migration window) + dr_mode 신규를 도입하여 ambiguity invariant 정합 + p99 회귀 차단.

### 1.1 직접 동기

MCT-169 (immediate local delete + ambiguity invariant, D3+D10) LAND 후 단계. NAS = single source of truth 확립의 reader 측 implementation:

1. **D7=A (Reader cache NFR)**: 95% hit + p99 <100ms. NAS-first reader 의 hot path 체감 지연 억제.
2. **D8=B (forward-only + local fallback)**: 새 데이터 NAS-first, cutover 이전 partition local fallback. migration window 동안만 유지 (D6=D sunset).
3. **dr_mode 신규**: NAS unreachable 시 state machine (CLOSED/OPEN/HALF_OPEN/UNKNOWN_TIER) + explicit mode flag override + Prometheus emit.

## §5 AC (7건)

- **AC-1 tier_reader priority chain**: `TierReader.read(partition_path)` → cache hit → NAS L1/L2/L3 → local fallback 우선순위 단위테스트 PASS (test_tier_reader.py)
- **AC-2 D7 NFR**: 10k read benchmark — hit_ratio ≥ 95% + p99 < 100ms (pytest-benchmark + prometheus)
- **AC-3 D8 local fallback**: NAS endpoint 차단 (dr_mode=OPEN) + cutoff 이전 partition → local Path read 응답 (ReadResult.status="local_fallback")
- **AC-4 reader_cache byte budget**: RSS delta ≤ 256 MB (10k entry 100 KB avg, byte-size LRU eviction verify)
- **AC-5 dr_mode state machine**: CLOSED→OPEN→HALF_OPEN→CLOSED 전이 + manual override (`set_mode("OPEN", reason="operator")`) + Prometheus `nas_reader_dr_state` emit
- **AC-6 backward compat**: MCT-154 기존 test 100% PASS preserve (cold_reader / reader_cache MCT-154 API / endpoint_router 전수 green)
- **AC-7 data NullReaderCache 제거**: `grep -r "NullReaderCache" mctrader-data/` = 0건 (LRUReaderCache 교체 완료)

## §6 INV (4건)

- **INV-1 SoT exclusivity preserve**: `tier_reader.read()` priority chain 이 D10 invariant (NAS+local 동시 존재 = violation) 위반 0. local fallback 진입은 NAS 부재 verify 후만.
- **INV-2 backward compat**: MCT-154 산출 API (ColdReader / ReaderCache MCT-154 method / EndpointRouter) 변경 0 — 기존 caller path 회귀 0.
- **INV-3 byte budget**: `ReaderCache.current_bytes()` ≤ `max_bytes` 상시 충족 (put() 후 invariant).
- **INV-4 D7 NFR baseline**: production 1h 측정 시 hit_ratio ≥ 95% AND p99 < 100ms. 미달 시 cache budget +50% 자동 escalate (Story 내 max 3 iter, FIX Ledger 박제).
