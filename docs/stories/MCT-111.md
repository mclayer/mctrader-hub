# MCT-111: code-review fix — mctrader-web

- **Status**: 구현 (fix 브랜치 생성 완료, follow-up 1건 + 테스트·merge 필요)
- **Trigger**: Codex 심층 리뷰 2026-05-09 — CRITICAL 2건 / HIGH 4건 발견
- **Repo**: `mctrader-web` (`c:\workspace\mclayer\mctrader-web`)

## 1. 배경

TLS 미적용(admin API plain HTTP), audit hash chain 파괴, idempotency schema mismatch,
rate limit bypass, RBAC 거부 미기록 등 보안·운영 결함 발견.
다섯 개의 fix 브랜치 + 1건 follow-up.

## 2. Fix 브랜치 현황

### `fix/web-tls`
| 항목 | 내용 |
|---|---|
| `api/config.py` | `get_uvicorn_tls_kwargs(host, dev=False)` 신규 — localhost/dev → ssl_*=None, 원격 → cert/key 경로 반환 |
| `api/cli.py` | `uvicorn.run(**tls_kwargs)` — ssl_certfile/ssl_keyfile 실제 전달, `--dev` 플래그 추가 |

### `fix/web-audit-retention`
| 항목 | 내용 |
|---|---|
| `scripts/cron_audit_retention.py` | 삭제 전 checkpoint hash 계산: SHA-256("pruned:{N}:{oldest_id}:{oldest_hash}:{ts}") |
| `scripts/cron_audit_retention.py` | 보존 체인의 최고령 row `prev_hash = "checkpoint:{hash}"` 업데이트 → 삭제 후에도 체인 검증 가능 |
| `api/admin/audit_db.py` | `verify_chain()`: "checkpoint:..." sentinel을 valid chain-start로 수용 |

### `fix/web-idempotency-schema`
| 항목 | 내용 |
|---|---|
| `api/admin/audit_db.py` | `idempotency_cache` DDL: `(key TEXT, endpoint TEXT, response_body TEXT, status_code INTEGER, created_at REAL, PRIMARY KEY(key, endpoint))` |
| `api/admin/idempotency.py` | DDL 통일, INSERT/SELECT 컬럼명 동기화, `endpoint` 파라미터 추가 (cross-action replay 차단) |

### `fix/web-rbac-ratelimit`
| 항목 | 내용 |
|---|---|
| `api/admin/rate_limit.py` | IP pre-auth bucket (10req/min per IP, `request.client.host`) — bearer prefix rotation bypass 차단 |
| `api/admin/auth_rbac.py` | `require_role()` 403 raise 전 `append_audit_row(outcome="rbac_reject")` (best-effort) |
| `api/admin/tokens.py` | `create_token()`: `expires_at` 기본값 `now + 30d`, DB insert에 전달 |

### `fix/web-alembic-0002`
| 항목 | 내용 |
|---|---|
| `alembic/versions/0002_alter_dispatch_alert_id_to_int.py` | upgrade(): non-numeric 행 SELECT → Exception (abort), out-of-range bigint → Exception (abort) 후 ALTER 실행 |
| 동 파일 | downgrade(): `postgresql_using="alert_id::text"` 정상 추가 |

## 3. Follow-up (이 Story 내 처리 필요)

### api/admin/rbac.py: CreateTokenRequest에 expires_at 미노출
현재 `tokens.py`의 `create_token()`은 `expires_at` 기본값(30d)을 사용하지만,
`rbac.py`의 `CreateTokenRequest` 모델이 해당 필드를 노출하지 않아 API 호출자가 만료 기간을 지정할 수 없음.

**수정**: `api/admin/rbac.py`의 `CreateTokenRequest`에:
```python
expires_at: datetime | None = None  # None → 30d default in create_token()
```
추가, `create_token(body.role, alias=body.alias, expires_at=body.expires_at)` 로 전달.
