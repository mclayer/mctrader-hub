# MCT-121: Upbit 거래소 데이터 수집 지원

- **Status**: 완료
- **Trigger**: Upbit KRW+USDT 시장의 OHLCV 백필 및 실시간 수집 지원 추가
- **Repos**: `mctrader-market-upbit` (신규), `mctrader-data` (멀티-exchange 확장)
- **PR**: [mclayer/mctrader-data#28](https://github.com/mclayer/mctrader-data/pull/28) — MERGED 2026-05-10

## 1. 배경

mctrader-data가 Bithumb만 지원하는 상태에서 Upbit KRW+USDT 시장의 OHLCV 백필(REST)과
실시간 수집(WebSocket)을 추가하고, 컨테이너를 거래소별로 분리하기 위해 작업.

## 2. 구현 범위

### Phase 1: mctrader-market-upbit (신규 리포)

| 모듈 | 내용 |
|---|---|
| `adapter.py` | `UpbitCandleProvider` — REST 백필, 200개/req 페이지네이션, `_verify_coverage` |
| `client.py` | `UpbitHttpClient` — 10 req/sec rate limit, 429/5xx 핸들링 |
| `ws_client.py` | `UpbitWebSocketStream` — 재접속+stale detection, exp backoff |
| `ws_events.py` | `UpbitTradeEvent`, `UpbitOrderbookEvent`, `UpbitTickerEvent` |
| `ws_mapping.py` | `normalize_message()` — Upbit raw JSON → typed event |
| `ws_subscribe.py` | `build_subscribe_message()` — ticket+channels 구독 payload |
| `ws_secret_guard.py` | public-only URL/헤더/payload 강제 검증 |
| `mapping.py` | Symbol ↔ market_code, Timeframe → REST 경로 |
| `exceptions.py` | 5 exception types |

- 테스트: 46개 (pytest), 모두 통과
- GitHub: https://github.com/mclayer/mctrader-market-upbit (private)

### Phase 2: mctrader-data 멀티-exchange 확장

| 파일 | 변경 |
|---|---|
| `src/mctrader_data/adapters.py` | 신규 — exchange → CandleProvider/WS stream 팩토리 |
| `src/mctrader_data/collector.py` | bithumb guard 제거, adapters 경유, kind-dispatch |
| `src/mctrader_data/cli.py` | backfill/collect `--exchange` 옵션, MetadataRefreshScheduler bithumb-only |
| `pyproject.toml` | mctrader-market-upbit git dependency 추가 |
| `compose.yml` | upbit-ingester 서비스 추가 (PORT 8081, NODE_UPBIT_A), 3-container 구조 |

## 3. 핵심 설계 결정

- **Symbol 변환**: Upbit market_code(`KRW-BTC`) = `str(Symbol)` → `symbol_to_market_code = str(s)` (zero 변환 코드)
- **Orderbook**: Upbit는 snapshot만 → `kind="orderbook_snapshot"`, orderbookdepth WAL ingester 미생성
- **Trade kind**: `kind="transaction"` (mctrader-data WAL 채널 키 일치)
- **MetadataRefreshScheduler**: Upbit Phase 2 연기 — `exchange == "bithumb"` guard
