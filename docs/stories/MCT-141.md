### 변경 내용

**MCT-132 framework 재사용**:
- MCT-132 Compactor Epic-A (LAND 2026-05-11) 의 L1/L2/L3 tiered compaction framework + ParquetWriter context manager pattern 답습
- spec/plan/runbook/Grafana dashboard 패턴 재사용

**Transaction-tier policy 신규**:
- 256 MB Parquet file roll (1000 ticks/sec 시 15-45분)
- 4-8 GB process limit
- partition layout: `market/transaction/schema_version=tick.v1.1/tier=L{1,2,3}/exchange={ex}/symbol={sym}/date={YYYY-MM-DD}/node={node}/part-{run_id}.parquet`
- 파일 폭발 시 fallback: `exchange/date/hour` + symbol column

**Storage upgrade (Story-5/6 open question)**:
- tick.v1 (8 col) → tick.v1.1 (11 col) Parquet schema 상승 — 본 Story 의 owner
- legacy tick.v1 row → v1.1 reader 가 신규 3 col NULL/default 처리 (backward compat)

**provenance column write**:
- `legacy_candle` (cutoff 이전 candle 파일) vs `transaction_derived` (cutoff 이후)

### Story-6 협조 항목 처리

- partition layout SSOT 정합 (v1 → v1.1 schema upgrade)
