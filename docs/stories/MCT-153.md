| **as_completed loop** | continuous (chunk count 만큼) | per-result checkpoint 갱신 + metric emit + evidence append | unconditional sequential (parallel dispatch 후 sequential collect) |
| **chunk timeout (EC-6)** | per-chunk (30s default) | `as_completed(timeout=...)` 호출 시 | conditional — 30s 초과 chunk 만 timeout exception, 다른 chunk 영향 0 (병렬 isolation) |
| **SOP recheck (graceful stop)** | per-result (Phase C step 9) | `if self._sop_runner.is_manual_gate(): executor.shutdown(...)` | conditional — SOP MANUAL_GATE 발생 시 graceful stop (cancel_futures=True), pending chunk 모두 sop_skipped 박제 |

**의도**: per-chunk thread 진입 = unconditional (모든 pending chunk 가 thread 진입 보장), per-thread 내 logic 만 conditional (PutResult/InvariantResult switch). EC-6 timeout 은 conditional (정상 chunk 영향 0).

#### §6.9.4 Quarantine retain placement (§6.1 chief decision enforcement)

- **rollback DELETE 거부 — 무조건 retain**: invariant FAIL 후 NAS object DELETE 0 (root cause 분석 차단 + RPO=0 위반 surface 차단). retain 자체가 unconditional.

- **재PUT 0 — unconditional**: retry budget loop 내 verify 만 retry, 재PUT 0 (HEAD-then-PUT idempotency 정합 + invariant FAIL 의 root cause 가 PUT 단계 아님).

#### §7.4.1 DR (Disaster Recovery) — backfill 운영 측면
