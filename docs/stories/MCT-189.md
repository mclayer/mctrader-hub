## §1 동기 (배경/문제)

사용자 운영 보고(2026-05-16): "로컬 디스크 용량이 계속 차고 있는데 S3에 데이터 제대로 적재되고 있는가."

진단 결과 = S3(NAS) 적재 자체는 정상이며, 디스크 압박의 원인은 **ADR-029 §D3=C grace-0 로컬삭제 정책이
코드에 wiring 안 된 상태**. EPIC-tier-promotion-single-source 가 2026-05-14에 POLICY_FINALIZED 박제
됐으나 §D3 핵심 deliverable(NAS HEAD verify → 즉시 로컬삭제) 의 실 production 경로가 미구현.

## §2 문제 (구체)

1. **`promote_l1()` 호출 site 0건** ([promotion.py:95](../../mctrader-data/src/mctrader_data/compactor/promotion.py))
   - 함수는 documented behavior 그대로 (HEAD verify ETag+VersionId → unlink wall-clock <100ms, HEAD 실패 시 local 유지)
   - 그러나 runner / l1 / l2 / l3 / dual_writer 어디서도 import·call 안 함
2. **DualWriter NAS PUT commit 후 source 로컬 삭제 로직 부재** ([dual_writer.py](../../mctrader-data/src/mctrader_data/nas_storage/dual_writer.py))
   - L1 NAS PUT `status=committed (ADR-029 D1=B)` 후에도 로컬 source parquet 보존
3. **L2/L3 (`l2.py`, `l3.py`) tier promotion 시 L1 source 로컬삭제 부재**
   - `idempotent skip` 로그가 누적 == NAS에 동일 sha256 존재해도 로컬 측 보존 지속
