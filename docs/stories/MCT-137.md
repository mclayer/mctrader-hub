### 변경 내용

**Aggregation Core Lib** (`mctrader_data/aggregation/core.py`):
- pure-Python aggregation core — Hot(asyncio) / Cold(DuckDB) 양 path 가 import
- 4 bar 알고리즘 SSOT:
  - **time bar**: 임의 timeframe, boundary `[start, end)` inclusion
  - **volume bar**: cumulative quantity threshold 도달 시 bar close
  - **tick bar**: cumulative trade count threshold 도달 시 bar close
  - **dollar bar**: cumulative price × quantity (KRW notional) threshold 도달 시 bar close
- 수치 표현: KRW scaled integer 또는 명시적 precision Decimal — boundary drift 방지
- per-symbol state: running OHLC + cumulative threshold accumulator + tie-breaking rule

**Contract Metadata** (`mctrader_data/aggregation/contract_metadata.py`):
- immutable metadata structure: `genesis_ts` / `threshold` / `precision` / `rounding_rule` / `source_cutoff` / `tie_breaking` / `version` / `contract_id` (SHA256)
- contract_id 발급 함수
- backward-compat versioning

### Lesson learned

- **결정성 강제 효과** — 4 bar aggregator 의 same input → same output 골든 replay test (6 tests) 가 핵심 trustworthiness 박제. Hot/Cold 양 path 가 동일 import 시 drift SLO < 0.01% (ADR-025 §D5) 보장 가능.
- **KRW scaled int boundary** — aggregation 내부 모든 누적 → scaled int (precision=0), persistence/Protocol boundary 에서 Decimal 변환. boundary 명시가 financial precision 핵심.
- **TDD + 결정론 fixture pattern** — 4 bar 종류 × edge case (정확 boundary / overshoot / tie-breaking) 로 deterministic invariant 강제.

### Open question (ADR-025 minor update 후보)

- **single-tick close ts_close drift rule** — overshoot 또는 threshold=1 시 ts_close == genesis_ts 충돌. 구현은 1µs advance (`state.genesis_ts + timedelta(microseconds=1)`) 적용 — deterministic + wall-clock 비의존. ADR-025 §determinism 에 1µs rule 박제 권장 (별도 Story 또는 Story-11 reconciliation harness 시점 amendment).
