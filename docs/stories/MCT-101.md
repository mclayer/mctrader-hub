### 11.2 Phase 4 발견 + 처리

#### 11.2.1 ADR-002 D6 reference correction

session prompt 의 "sqlite (event store, ADR-002 D6)" 명시가 incorrect (ADR-002 D6 = mctrader-engine paper ledger, mctrader-web sqlite = ADR-016 audit_log). 코드 reading 시 정정 → ADR amendment path 도 ADR-016 단독 (D12 γ) 으로 결정.

#### 11.2.2 TLS localhost vs Docker bind 충돌 (D6 invariant)

config.py `validate_tls_for_host()` 가 non-localhost binding 시 TLS 의무. compose-internal `0.0.0.0` bind → TLS 부재 → 기존 invariant 위반. D6 채택 = env exempt path (`MCTRADER_ALLOW_NON_LOCALHOST_NO_TLS=1`) + host port 미노출 의무 (3-asset enforcement: compose.yml + README + CI gate test_compose_invariant.py).

#### 11.2.3 Cross-stack volume namespacing (D8 invariant)

mctrader-data Pilot project name = `mctrader-data` → volume = `mctrader-data_mctrader_data`. compose `external: true` reference. fail-loud invariant 박제 (smoke 8.B negative case). standalone fallback `MCTRADER_DISABLE_DATA_STATUS=1` env.

#### 11.2.4 Pre-existing pyright failure (MCT-97 P6 era, fixed during PR)

origin/main CI 가 pyright 단계에서 8 errors 로 fail 중 (P5 era 의 4 fixture 의 `-> AsyncClient` annotation 누락 — async generator 가 yield 사용). PR #22 가 inherit 후 root fix (commit `e71bcf5`) — `AsyncGenerator[AsyncClient, None]` 으로 수정. MCT-101 scope 외 작업이지만 `escalate-and-fix` 정신 (consumer workaround 금지) 정합.

#### 11.2.5 Pre-existing collector e2e test failure (NEW finding, MCT-97 P6 era)

pyright fix 후 pytest 단계 도달 시 `tests/e2e/test_admin_xplat_linux.py::test_linux_systemd_collector_start` Linux-only e2e test FAIL (returns 500 instead of 200). 동일 file 의 paper_runner / backtest / wfo / health smoke 는 PASS — collector start verb 만 특화 issue.

분석:
- Linux CI 환경 특화 (Windows local 에서는 `sys.platform != 'linux'` skip)
- monkeypatch on `adapter_mod.dispatch` 이 collector path 에 효과 안 미치는 가능성 (import binding pattern)
- pyright failure 가 pytest 도달 막아 기존에는 hidden
