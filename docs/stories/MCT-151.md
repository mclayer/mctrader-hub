### 도메인 용어 정합

- **dual-write atomic primitive**: 양쪽 저장소 (local volume + NAS MinIO) 동시 PUT + 2-phase commit semantic. caller (compactor / future MCT-152 dual_write_window_runner) 가 본 primitive inject 후 source 삭제 결정. 본 Story = primitive 정의만, 실 caller 통합은 MCT-152 scope. ADR-027 D4 cutover step 1 (dual-write window) 의 atomic guarantee.
- **L1 compaction drain barrier (S2)**: drain = L1 compactor 측 in-flight task 모두 완료 wait (signal-based, polling 또는 event hook). barrier = 신규 compaction task 시작 차단 (compactor pause signal emit, ADR-027 D5 hot path 무영향 정합 — collector WAL/L1 ParquetWriter 영향 0). 본 barrier 가 dual-write toggle 활성화 직전 적용 → toggle 시점 compaction output 이 local 에만 land 하는 silent row loss 차단.
- **2-phase commit semantic**: phase 1 = 양쪽 prepare (local tmp write + NAS PUT), phase 2 = atomic visibility (양쪽 commit 또는 양쪽 rollback). 본 Story 의 single segment write 단위 — DB transaction semantic 의 abstract 적용. NAS-side `hard_floor_blocked` (MCT-150 §6.7 박제) 시 commit/rollback 결정 분기 = ArchitectPL §6 결정.
- **7종 invariant (S5 박제 — ADR-027 D6 amendment)**:
  1. **sha256** (D6 source) — local L2/L3 Parquet sha256 == NAS object sha256 (full byte-level identity)
  2. **object count** (D6 source) — local partition file count == NAS bucket object count (per partition)
  3. **parquet row count** (D6 source) — local Parquet row count == NAS object 다운로드 후 parquet row count (per file)
  4. **column count** (S5 신규) — local Parquet column count == NAS object column count == ADR-009 16
  5. **column name order** (S5 신규) — local schema column name 순서 == NAS object schema 순서 == ADR-009 §D2 정의
  6. **dtype identity** (S5 신규) — local Parquet pyarrow type-level identity == NAS object dtype (Decimal precision/scale 포함)
  7. **schema_version pin** (S5 신규) — local Parquet `schema_version=v1` partition prefix == NAS object prefix (legacy schema 침범 0)
- **invariant harness**: `invariant_harness.verify()` single-shot — single segment 또는 partition batch 단위. PASS/FAIL per invariant + 측정값 emit (Prometheus `nas_invariant_*` namespace).
- **chaos test**: pytest framework 내 NAS endpoint 단절 + L1 in-flight 동시 시뮬레이션. `responses` 또는 mock fixture 활용 (MCT-150 PoC pattern 참조). RPO=0 invariant 보존 검증 — segment drop 0 + 7종 invariant ALL PASS 또는 명시적 FAIL surface.
- **`nas_invariant_*` metric prefix**: MCT-150 `nas_uploader_*` prefix-disjoint enforce. 예: `nas_invariant_sha256_match_count` (Counter, labels: partition) / `nas_invariant_schema_drift_count` (Counter, labels: partition, drift_type) / `nas_invariant_verify_latency_seconds` (Histogram). 정확 metric naming = ArchitectPL §6 결정.

## §5 통합 요구사항 명세

### §5.1 Acceptance Criteria (Issue #257 body 박제, Given/When/Then 확장)

**AC-1**: dual_writer.write() atomic primitive (local + NAS 동시 PUT, 2-phase commit semantic)

- **Given** MCT-150 NASUploader 가 가용 + local volume mount 가용
- **When** `dual_writer.write(local_path, nas_key, data, sha256)` 호출
- **Then**
  - phase 1 (prepare): local tmp write + `nas_uploader.put(nas_key, data, sha256=sha256)` 호출
  - phase 2 (commit/rollback): 양쪽 prepare 성공 시 → atomic visibility (local rename + NAS object visible) + caller source 삭제 가능 status return. NAS-side `hard_floor_blocked` 시 → caller source retain status return + local-side write 결정 분기 (ArchitectPL §6 결정: rollback or retain or quarantine)
  - PutResult.status 5종 enum (MCT-150 §6.7 박제) 그대로 propagation — 특히 `hard_floor_blocked` 시 caller 가 source retain 의무
  - forward-only invariant (ADR-009 §D12.2) 보존 — 양쪽 신규 row append-only
  - ADR-017 hot path 무영향 — collector WAL/L1 ParquetWriter 침범 0

**AC-2**: compaction_barrier.drain_and_block() (L1 compaction drain + barrier, dual-write toggle gate)

- **Given** L1 compactor 가 in-flight task 보유 가능
- **When** `compaction_barrier.drain_and_block()` 호출 (dual-write toggle 활성화 직전)
- **Then**
  - drain: L1 compactor 측 in-flight task 모두 완료 wait (signal-based, ArchitectPL §6 결정 mechanism)
  - barrier: 신규 compaction task 시작 차단 (compactor pause signal emit)
  - ADR-017 hot path 무영향 — collector WAL append + L1 ParquetWriter fsync 영향 0
  - barrier 적용 후 dual-write toggle 활성화 시점 → compaction output 이 local 에만 land 하는 silent row loss 차단 (S2 박제)
  - barrier 해제 (`compaction_barrier.release()`) 호출 시 신규 compaction 재개 가능

**AC-3**: invariant_harness.verify() (7종 invariant ALL PASS, single-shot)

- **Given** local partition + NAS partition 가용 (양쪽 segment write 완료 후)
- **When** `invariant_harness.verify(local_partition, nas_partition)` 호출
- **Then**
  - 7종 invariant 검증 (ALL PASS 시 PASS, 1종이라도 FAIL 시 FAIL, ArchitectPL §6 결정 — D6 본문 "1종이라도 FAIL 시 cutover 차단" 정합 base):
    1. sha256: local Parquet sha256 == NAS object sha256 (per file)
    2. object count: local partition file count == NAS bucket object count (per partition)
    3. parquet row count: local Parquet row count == NAS object row count (per file)
    4. column count: local + NAS column count == 16 (ADR-009 §D2.1)
    5. column name order: local + NAS column 순서 == ADR-009 §D2 정의
    6. dtype identity: local + NAS pyarrow type-level identity (Decimal precision/scale 포함)
    7. schema_version pin: local + NAS partition prefix `schema_version=v1`

@dataclass(frozen=True)
class DualWriteResult:
    """Dual-write atomic primitive 의 result enum + caller contract.

    status enum 3종 (FIX#4 P1-NEW-3 wording SSOT 정합 — single string 박제):
    - "committed":            local + NAS 양쪽 atomic visible (양쪽 prepare 성공 → atomic commit).
                              caller source 삭제 가능 (양쪽 영속화 보장, RPO=0).
    - "local_only":           local prepare 성공 + NAS PUT 실패 시 NASUploader.put() 가
                              PutResult.status="queued" return (retry_queue persistent enqueue).
                              local-side 는 atomic visible. caller source 삭제 가능 (retry_queue
                              가 후속 drain 책임). MCT-150 §6.7 caller contract 직접 정합.
    - "hard_floor_blocked":   NASUploader.put() 가 PutResult.status="hard_floor_blocked" return
                              (retry_queue Stage 1 unconditional guard 차단, MCT-150 §6.2.2 박제).

                              **caller source retain 의무** (RPO=0 보존, S8 user_confirmed).

    nas_put_result: NASUploader.put() 의 raw PutResult propagate (debugging 용 — caller switch
    는 status 만 사용).
    """

    nas_put_result: PutResult
    local_path: Path
    nas_key: str
    sha256: str
    latency_ms: float


class DualWriter:
    """Dual-write atomic primitive — local + NAS 동시 PUT, 2-phase commit semantic.

    Responsibilities:
    - phase 1 (prepare): local tmp write (rename 보류) + NASUploader.put(suppress_enqueue=False).
    - phase 2 (commit/rollback): NASUploader PutResult.status switch 후 local-side 결정.
        - status="uploaded" or "skipped_idempotent" or "skipped_etag_overwrite"

        - status="hard_floor_blocked" (retry_queue hard floor 차단)

    ADR-017 hot path 무영향: collector WAL append + L1 ParquetWriter fsync 영향 0 (별 process /
    별 file path). dual_writer 가 L3 compaction tier 의 cold tier 측 callsite 만 변경, hot path

    inject 후 source 삭제 결정 — 본 Story scope = primitive 정의만, 실 caller 통합은 MCT-152 scope.

    - DualWriteResult.status ∈ {"committed", "local_only"} → 영속화 보장 → caller source 삭제 가능
    - DualWriteResult.status == "hard_floor_blocked" → 양쪽 영속화 0 → caller source retain 의무

    def __init__(
        self,
        nas_uploader: NASUploader,
        local_root: Path,                    # local L2/L3 tier 의 base path (tier=L2|L3 partition prefix)
        metrics: "InvariantPrometheusExporter | None" = None,  # 본 Story 신규 (§6.2.4)
    ) -> None:
        ...

    def write(
        self,
        *,
        local_path: Path,                    # 최종 local rename target (local_root 하위 partition)
        nas_key: str,                        # NAS object key (Hive prefix `schema_version/exchange/symbol/date/node=<id>/tier=L{2,3}/...`)
        data: bytes | Path,                  # segment payload (bytes 또는 source file path)
        sha256: str,                         # data 의 sha256 hex (caller 측 사전 계산 의무, idempotency key)

        Phase 1 (prepare):

            tmp_path = local_path.with_suffix(local_path.suffix + ".tmp")

            **caller source retain 의무** (양쪽 영속화 0, source 삭제 시 RPO=0 위반 — S8 user_confirmed).

        Caller contract (§6.7 cross-module contract 표 박제):
        본 method 의 return 후 caller 가 status 검사 후 source 삭제 결정 의무. status 미검사 +
        무조건 source 삭제 시 → "hard_floor_blocked" case 에서 RPO=0 위반 (silent loss).

        Metrics emit (선택, metrics != None 시):

        - nas_invariant_dual_write_local_only_count (Counter, labels: nas_key prefix)

        가 1차 방어선 (status="skipped_idempotent" return) → forward-only 위반 0. dual_writer

        sha256 mismatch 처리 — MCT-150 §6.2.1 박제 정합).

    drain_wait_ms: float
    in_flight_remaining: int


class CompactionBarrier:
    """L1 compaction drain + barrier — dual-write toggle gate (S2 박제).

    Responsibilities:
    - drain: L1 compactor 측 in-flight task 모두 완료 wait (signal-based, polling 또는 event hook).
      ArchitectPL §6 결정 mechanism = **signal-based polling** (default, EC-2 박제 결정):
      L1 compactor 가 in-flight task count 를 shared state (e.g., process file `/data/compaction_state.json`

    - drain wait = L1 compactor 측 신호 수신만 (collector WAL append + L1 ParquetWriter fsync

    - 본 barrier 적용 중에도 collector WAL append + L1 ParquetWriter fsync 정상 동작 — collector

    본 Story 의 barrier 는 signal emit 만 — 실 L1 compactor hook 통합은 MCT-152 dual_write_window_runner

        ...

    def drain_and_block(self) -> BarrierResult:
        """L1 compaction drain + barrier 적용 (single-shot, dual-write toggle 직전 호출).

        Algorithm:
        1. emit barrier signal: write to barrier_signal_path (L1 compactor 측이 watch 후 신규 task spawn 차단).
        2. drain wait loop (polling, signal-based):
            start_ts = monotonic_now()
            while monotonic_now() - start_ts < drain_timeout_seconds:
                state = read compaction_state_path (in_flight_count + last_update_ts)
                if state.in_flight_count == 0:
                    return BarrierResult(status="ok", drain_wait_ms=elapsed,
                                         in_flight_remaining=0)
                sleep(poll_interval_ms / 1000)
            # timeout 도달
            state = read compaction_state_path
            return BarrierResult(status="drain_timeout", drain_wait_ms=elapsed,
                                 in_flight_remaining=state.in_flight_count)

        Return enum semantics (BarrierResult.status):
        - "ok": drain + barrier 정상 → dual-write toggle 활성화 가능.
        - "drain_timeout": timeout 도달 → **dual-write toggle 활성화 차단 의무** (silent local-only
          land 차단). caller 가 release() 호출 후 manual gate 진입 + alert.

          caller 가 즉시 release() + emergency rollback. **본 status 는 release() 후 verify
          step 에서 검출** — drain_and_block() 자체에서는 "ok" or "drain_timeout" 만 return.
          (별 verify method `verify_barrier_intact()` 를 caller 가 호출 — Phase 2 DeveloperPL 결정,
          본 Story design intent 박제만.)

        Idempotency: 본 method 는 idempotent — 이미 barrier 적용 상태에서 호출 시 drain wait 즉시
        return "ok" (in_flight_count=0 검출). signal_path 의 write 도 idempotent (동일 content
        overwrite).

        Hot path 무영향 invariant 검증: 본 method 실행 중에도 collector WAL append + L1 ParquetWriter
        fsync 정상 동작 — collector 측 process 와 본 barrier 가 별 process / 별 file path. AC-2
        chaos test (§8.2) 가 hot path 무영향 검증.

        Metrics emit:
        - nas_invariant_compaction_barrier_drain_wait_seconds (Histogram)
        - nas_invariant_compaction_barrier_status_count (Counter, labels: status enum)
        """

    def release(self) -> None:
        """Barrier 해제 — 신규 compaction 재개 가능.

        Algorithm: barrier_signal_path 삭제 (또는 content "released" overwrite). L1 compactor
        측이 signal 부재 검출 후 신규 task spawn 재개.

        Idempotent: 이미 release 상태에서 호출 시 NO-OP.

        Caller 의무: drain_and_block() return 후 dual-write toggle 종료 시점에 release() 호출
        의무. release 누락 시 L1 compaction 영구 차단 → hot path 영향 (L1 → L2 promotion 중단).

### §11.3 7종 invariant 의 layer 별 검증 (byte-level / set-level / schema-level)

| Layer | Invariant | 측정 source | mismatch 시 status |
|---|---|---|---|
| **byte-level (1종)** | sha256 | local Parquet sha256 (per file) == NAS object sha256 (per file, NAS HEAD response Metadata 또는 GET 후 hash) | `sha256_fail` |
| **set-level (2종)** | object_count | local partition file count == NAS bucket object count (per partition prefix) | `object_count_fail` |
| **set-level (2종)** | row_count | local Parquet row count == NAS object row count (per file, pyarrow ParquetFile.num_rows) | `row_count_fail` |
| **schema-level (4종)** | column_count | local + NAS column count == 16 (ADR-009 §D2.1, pyarrow Schema.names length) | `column_count_fail` |
| **schema-level (4종)** | column_order | local + NAS column 순서 == ADR-009 §D2 정의 (pyarrow Schema.names list 정확 일치) | `column_order_fail` |
| **schema-level (4종)** | dtype | local + NAS pyarrow type-level identity (Decimal precision/scale 포함, type-level 비교) | `dtype_fail` |
| **schema-level (4종)** | schema_version | local + NAS partition prefix `schema_version=v1` (string match) | `schema_version_fail` |

**Layer 별 검증 의도 박제**:

- **byte-level (1종)**: D6 본문 직접 source — full byte identity 보장 (compression / corruption 검출).
- **set-level (2종)**: D6 본문 직접 source — file 단위 누락/중복 검출 (object_count) + 내부 row 단위 누락/중복 검출 (row_count).
- **schema-level (4종)**: S5 신규 박제 — D6 PASS 이나 Parquet schema 차이로 read 파괴/오염 risk 차단 (reader-breaking drift 포착). reader 측 분기 코드 (e.g., `if "volume" in schema:` 또는 `df["close"].astype(Decimal)` ) 의 silent break 차단.

### §11.6 Idempotency (DataMigrationArch primary + OperationalRiskArch consult — CFP-46 CONDITIONAL)

**Idempotency 요건 평가**: 본 Story = dual-write atomic primitive + InvariantHarness verify (read-only) + CompactionBarrier signal emit → idempotency invariant 필수 (특히 DualWriter retry cycle + InvariantHarness 매일 cron 반복 호출).

- **DualWriter idempotency primitive (MCT-150 NASUploader.put() HEAD-then-PUT inject)**:
  - DualWriter.write() 가 NASUploader.put() 직접 inject — NASUploader 의 HEAD-then-PUT idempotency (S4) 가 1차 방어선
  - 동일 (nas_key, sha256) tuple 에 대해 반복 DualWriter.write() 호출 시 → NASUploader status="skipped_idempotent" return → DualWriter status="committed" propagate (forward-only 위반 0)
  - sha256 mismatch 시 NASUploader ConditionalWriteConflict raise → DualWriter 가 raise propagate → caller 폐기 또는 SOP escalate
- **idempotency key 정의**: (bucket, nas_key) tuple + sha256 hash. backfill (MCT-153) 의 per-(symbol, day) tuple key 와 정합. local rename target (local_path) 도 동일 path 반복 시 OS atomic rename 의 idempotent 정합.
- **CompactionBarrier idempotency primitive**:
  - drain_and_block() 가 idempotent — 이미 barrier 적용 상태에서 호출 시 drain wait 즉시 return "ok" (in_flight_count=0 검출). signal_path 의 write 도 idempotent (동일 content overwrite).
  - release() 가 idempotent — 이미 release 상태에서 호출 시 NO-OP (signal file 부재 시 unlink missing_ok=True).
- **InvariantHarness idempotency primitive**:
  - verify() = read-only invariant — 동일 (local_partition, nas_partition) 입력에 대해 반복 호출 시 결과 일관 (storage 변경 0)
  - per-segment streaming hash 또는 NAS metadata sha256 (HEAD response Metadata) 모두 read-only — idempotent 정합
- **Side-effect-free retry (DualWriter chaos case)**: NAS unreachable 시 DualWriter.write() phase 1 NAS PUT → NASUploader retry_queue.enqueue() 자동 호출 → status="queued" return → DualWriter phase 2 local rename 후 status="local_only" return. 후속 재호출 시 retry_queue 가 후속 drain 책임 — DualWriter 측에서 재enqueue 0 (이중 enqueue 차단, MCT-150 §11.6 P0-2 suppress_enqueue 패턴 정합).
- **Hard floor case idempotency (caller source retain)**: DualWriter status="hard_floor_blocked" return 시 caller 가 source retain (RPO=0 보존). SOP reset 후 caller 가 동일 (nas_key, sha256) tuple 재호출 시 → NASUploader HEAD-then-PUT 가 1차 방어선 (sha256 match 시 skip, mismatch 시 ConditionalWriteConflict).
