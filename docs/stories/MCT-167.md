# MCT-167 — Cold tier governance v2 singleton (ADR-029 신규 + 3 amendment + DR runbook stub)

## §1 동기

EPIC-tier-promotion-single-source 의 **governance singleton Story** — ADR-029 (Cold tier governance v2 — NAS = Single Source of Truth for ALL tiers) 신규 publish + ADR-017/027/009 amendment 3건 박제 + DR runbook Epic-level scope 확장 (stub).

### 1.1 직접 동기

MCT-156/162/160 3-cycle 누적 실패 patterns 의 근본 원인 = **local-NAS dual-storage ambiguity**. 현재 cold tier (L2/L3) 만 NAS 격상 + hot tier (WAL/L1) = local only. read path 측 "어디까지가 진실의 source 인지" 모호 → review evidence 가 production 실측을 lag (review lane PASS vs production 실측 결함).

사용자 directive 2026-05-14 (autonomous 자율 진행):

1. **L1 도 NAS dual-write** — collector WAL → L1 ParquetWriter atomic 직후 NAS PUT 의무. ADR-027 §D5 "L1 NAS upload 금지" invariant 폐기.
2. **상위 tier promotion 후 local delete** — NAS = single source of truth, local = ephemeral cache only.
3. **ambiguity 차단** — dual-storage 의 source authority 모호 차단.
4. **WAL Local 유지** — WAL = local only (hot path zero-loss, ADR-017 정합).
5. **로컬 용량 제한** — WAL 30 GiB + L1 20 GiB + NAS 500 GiB target + host 200 GiB hard limit.
