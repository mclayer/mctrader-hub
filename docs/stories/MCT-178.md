# MCT-178 — backtest-runner profile + oneshot + compose config CI lint + signal-collector Redis migration

## §1 동기

**EPIC-mctrader-docker-stack Story-4 (sequential_phase 4)** — MCT-177 LAND (hub#333 + data#65 +
engine#54 + hub#334 + Phase 2 PR2) 로 `mctrader-engine paper-engine` daemon 이 완성됨. 본 Story 는
그 위에 두 번째 engine service — `backtest-runner` 를 oneshot profile 로 추가하고, compose config CI
lint workflow 를 신규 박제한다.

4 결정 구현 + MCT-177 carry over 1 항목:

1. **D2 backtest-runner (oneshot profile)** — compose.yml 에 `backtest-runner` service 신규
   (image: ghcr.io/mclayer/mctrader-engine, profiles: [oneshot], command: `backtest ...`,
   restart: "no", no healthcheck). paper-engine 와 동일 image, command 만 분기.

2. **D4 oneshot completion** — backtest 1회 실행 후 exit 0 → 컨테이너 종료 (`restart: "no"` 정합).
   SIGTERM = 진행 중 backtest graceful abort (기존 `shutdown.py` asyncio SSOT 경유 — MCT-177 LAND).
   healthcheck 없음 (one-shot 컨테이너 성격 정합).

3. **D10 universe override** — `docker compose run --rm backtest-runner backtest --universe-id <id>`
   case-specific override (MCT-177 LAND CLI option 재사용).

4. **D16 compose config CI lint** — `.github/workflows/compose-validate.yml` 신규 —
   PR push 시 `docker compose --profile dev/prod/oneshot config --quiet` lint +
   `docker compose --profile dev up -d --wait` health gate (3분 budget).

5. **signal-collector Redis migration (MCT-177 carry over)** — signal-collector 5종
   (fear_greed / ecos / kimchi / announcement / coinglass) Redis key 를 unprefixed → `signal:*` rename.
   1주일 dual write (legacy + prefixed 동시 write) + Prometheus `redis_key_migration_dual_write_active`
   Gauge. LAND+7d legacy cleanup 별 PR.

## §6 Risk 상세

### R-MCT-178-1 compose-validate workflow CI budget 초과 (MEDIUM)

- **위협**: `docker compose --profile dev up -d --wait` health gate 가 3분 budget 초과 →
  PR CI FAIL (infra 기동 slow, postgres/minio/redis cold start)
- **mitigation**:
  - `--wait-timeout 180` 명시 (3분 budget)
  - health gate = infra only (postgres + redis + minio) — 어플리케이션 service 제외
  - 초과 시 `compose down` cleanup + CI log 박제
- **잔여 risk**: CI runner (ubuntu-latest) 의 resource 제약 → MCT-180 testcontainers 로 보완

### R-MCT-178-2 signal-collector dual write silent fail (MEDIUM)

- **위협**: signal-collector 5종 중 일부가 prefixed write 실패 시 paper-engine `signal:*` read miss →
  silent data hole (MCT-177 §6 R-MCT-177-1 동형)
- **mitigation**:
  - dual write = legacy unprefixed + `signal:*` 동시 write (개별 Redis SET, 각자 RESP 확인)
  - Prometheus `redis_key_migration_dual_write_active` Gauge (1=활성) — 비활성 시 alert
  - 실패 시 caller 측 exception 로그 + Gauge 0 → operator 알림
- **잔여 risk**: Redis container crash 시 양방 write 모두 소실 — Redis AOF = MCT-179 결정

### R-MCT-178-3 compose-validate profile 미매칭 (LOW)

- **위협**: `--profile oneshot` 로 config lint 시 backtest-runner service 부재 (profiles 설정 오류)
  → CI PASS 했으나 실 실행 FAIL
- **mitigation**:
  - `docker compose --profile oneshot config --quiet` lint 별도 step (AC-4)
  - AC-1 에서 `docker compose --profile oneshot run --rm` 실 실행 verify
