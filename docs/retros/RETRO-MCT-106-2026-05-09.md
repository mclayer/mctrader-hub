### 2.3 F10 `compaction_run_id` determinism — timestamp 제거로 same-partition uniqueness invariant 박제

Iteration 1 F10 (P1, implementability) 의 정확한 진단 — 기존 `compaction_run_id = sha256(sealed_path) + timestamp` 정의는 **재처리 시 다른 filename → 동일 partition multiple file** 발생 가능. ArchitectPL FIX 시:

```
compaction_run_id = sha256(sealed_segment_relative_path).hexdigest()[:16]
                                                          (timestamp portion 제거)
```

**Same-partition uniqueness invariant 신규 박제**: 동일 (`tier`, `exchange`, `symbol`, `date`, `node=`) partition 안에서 동일 `compaction_run_id` = 동일 source sealed segment. 충돌 detected 시 fail-closed halt + ledger emit. tmp 파일명에는 `os.getpid()` 포함 (`part-{run_id}.{pid}.tmp`) — write race 회피.

**Step 5 collision 처리 case A/B 분리** (legacy ambiguity 해소):
- case A: 코드 변경 (column ordering 등) → `code_change_detected` sub-reason
- case B: 비결정적 처리 → bug report (deterministic invariant violation)

→ **idempotency invariant 의 정수**. INV-3 (idempotency) 와 §11.6 (data migration idempotency) 가 단일 SSOT 로 수렴. 향후 다른 compactor / batch processor 류 Story 의 reference pattern.
